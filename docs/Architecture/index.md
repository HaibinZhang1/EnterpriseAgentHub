# P2 Architecture Assets

- [Domain Boundaries](domain-boundaries.md)

## How to use this directory

- Read `overview.md` first when onboarding.
- Read `domain-boundaries.md` before adding new modules or refactoring ownership.
- Read `layering-rules.md` before moving files or introducing new imports.
- Read `shared-contracts-rules.md` before changing any API/Desktop shared types.
- Read `extension-points.md` before adding new tool adapters, review rules, or notification sources.
