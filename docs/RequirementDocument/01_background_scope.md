# 1. 项目背景与目标

## 1.1 项目背景
企业内部会逐步沉淀大量面向不同 AI 工具、项目场景、部门业务的 skills。当前缺少一个统一的内部市场来完成以下事情：

- skill 的统一发布、审核、上架、下架、归档
- skill 的统一搜索、浏览、安装、启用、更新
- skill 的统一权限控制、部门范围管理、可见范围管理
- skill 的统一治理，包括审核、排行榜、版本管理、操作追踪

## 1.2 项目目标
搭建一个可在企业内网部署的 **Agent Skills 管理市场（Marketplace）**，让企业内部用户和管理员围绕 skill 完成以下闭环：

- 明确 skill 的定义、元信息规范与包结构规范
- 统一管理 skill 的发布、审核、展示、安装、启用与更新
- 支持按部门、权限、可见范围进行精细化控制
- 提供面向普通用户、发布者、审核员、管理员的统一操作入口
- 为后续扩展 MCP、插件管理、Agent 检索等能力预留空间

## 1.3 项目定位
本系统不是单纯的文件上传站，而是企业内部的 **skill 目录 + 审核中心 + 使用管理中心 + 治理平台**。

# 2. 范围与边界

## 2.1 当前版本 Desktop 一期范围
当前版本 聚焦 Desktop/Tauri 桌面客户端的使用闭环，目标是让企业用户可以发现、安装、启用并管理自己本机的 Skill。当前正式安装包交付仍以 Windows exe 为主；macOS 已纳入 Desktop/Tauri 本地能力的编译、运行和测试支持，但暂不交付 macOS 安装包；服务端必须部署到 Linux 服务器，优先选择简单、兼容性高的方案。

- 登录与身份区分
- 服务端连接状态展示与异常提示
- 市场浏览与详情查看
- 市场搜索、筛选、排序的基础能力
- 固定中文短分类 / 标签体系：分类必选 1 个，标签选择 1 到 5 个
- 我的已安装 Skill 管理
- skill 安装、更新、卸载
- 工具与项目维度的 skill 启用管理
- 通知（应用内消息中心）与基础状态提示
- 基础设置
- Skill 包结构与元数据规范
- 离线场景基础支持：已安装 Skill 可继续使用，可在本地启用/停用，市场搜索和安装/更新不可用
- 内置支持 Codex、Claude、Cursor、Windsurf、opencode、自定义目录，并做真实格式转换
- 启用方式采用 symlink 优先；若权限、文件系统或安全策略不允许，则自动降级为 copy，并记录降级原因
- 单个 Skill 包最大 5MB，包内文件最多 100 个

> 当前版本 账号边界：使用自建账号体系，由管理员开通账号；手机号是唯一登录凭证，格式为 1 开头的 11 位数字；用户名称对外展示且允许重名，内部账号 ID 不对外展示；后续再考虑接入企业系统或统一身份源。

## 2.2 后续版本 企业治理闭环
后续版本 补齐企业内部发布、审批、管理与权限治理：

- **自建账号体系**（系统初始一个一级管理员，用户新增和角色由上一级管理员设置）
- skill 发布与版本更新
- 审核流程与审核工作台
- 权限变更审核
- 部门、用户、skill 管理
- 下架 / 上架 / 归档
- 审核模板、拒绝原因模板与风险标签枚举
- manifest.json 最终字段与枚举值固化

## 2.3 预留版本 生态与规模化扩展
预留版本 面向更大范围的生态接入、运营治理和组织复杂度：

- MCP 管理能力
- 开源 AI 工具插件管理
- 智能 agent 搜索
- RAG 检索增强
- 自动替用户完成卸载后的环境修复
- 审核通过后的高风险追溯复核机制
- 审核员离线 / 离职后的复杂接管机制
- 组织与人员变更后的自动迁移编排
- 审计日志
- 批量用户导入/管理
- 数据统计看板
- 多维护者/协作模型
- 通知推送渠道集成
- Skill 依赖自动解析
- 批量 Skill 目标管理

## 2.4 当前版本 明确不做
以下内容不进入 当前版本 Desktop 正式交付范围：

- skill 发布与版本更新表单
- 审核流程与审核工作台
- 部门、用户、skill 管理台
- Web 管理台或独立 Web 发布入口
- 完整系统安全扫描能力（当前版本 只展示服务端提供的风险/状态字段，不做真实扫描实现）
- 风险脚本扫描
- macOS、Linux 安装包或 exe 之外的安装包（不含 Desktop/Tauri 运行时适配与本机验证能力）
- 企业统一身份源或外部账号系统接入
- 分类与标签体系治理（当前版本 可展示已有标签/分类，不负责体系定义）
- 企业 IM、系统托盘等外部通知渠道
