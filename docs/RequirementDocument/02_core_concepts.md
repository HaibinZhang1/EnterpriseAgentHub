# 3. 核心概念定义

## 3.1 Skill 定义
Skill 指一个可被企业内部用户发现、审核、安装、启用、版本化管理和权限控制的能力单元。
Skill 的交付形式为文件夹或 zip 包，系统以 skill 为最小管理对象进行治理。

## 3.2 Skill 的关键权限维度

### 3.2.1 授权范围
定义谁**可以安装、下载、启用和使用**该 skill。

可选范围：
- 本部门
- 本部门及下级部门
- 指定多个部门
- 全员可用

### 3.2.2 公开级别
为避免"公开"语义不清，系统统一使用以下三个标准级别：

- **摘要公开**：非授权用户可看到 skill 的摘要信息，但不可查看完整详情，不可安装
- **详情公开**：非授权用户可查看完整详情，但仅授权范围内用户可安装/启用
- **全员可安装**：所有用户都可查看详情并安装/启用

> 说明：
> 1. "公开"不再作为单独词汇使用，全部收敛为"摘要公开 / 详情公开 / 全员可安装"三种标准级别。
> 2. "全员可安装"同时意味着全员可见详情。
> 3. 若未显式配置公开级别，则默认不公开，仅授权范围内可见。

### 3.2.3 展示规则映射
- 非公开：仅授权范围内可在市场、搜索、排行榜、详情页中看到
- 摘要公开：非授权用户可看到摘要卡片与受限详情页
- 详情公开：非授权用户可进入完整详情页，但安装/启用按钮置灰
- 全员可安装：全员完整可见、可安装、可启用

## 3.3 安装与启用

### 基本定义
- **安装**：Skill 从服务端下载到本地 Central Store（唯一真源），但未必正在生效
- **启用**：Skill 通过 copy 方式从 Central Store 分发到指定工具或项目的 skills 目录中生效

### 技术实现参考
启用操作的技术实现参照 [skills_manage.md](skills_manage.md) 中的 Central Store + Tool Adapter 架构：
- 每个 AI 工具有一个 Tool Adapter，定义路径、格式、安装方式
- P1 固定使用 copy，不使用 symlink
- 工具检测方式：**注册表扫描 + 路径探测**

### 关键规则
- 一个 skill 可同时启用到多个工具和多个项目
- 同一个 skill **不允许多版本并存**（粒度为**每台设备本地只能有一个版本**）
- 安装需通过**服务端中转**下载
- 更新时服务端提供 Hash，客户端做全量覆盖

## 3.4 Star
Star 的业务含义为：
- 收藏
- 推荐
- 提高搜索排序权重

其含义类似 GitHub Star，统计口径为**实时生效数量**：
- 用户 Star 一次计 +1
- 取消 Star 计 -1
- 再次 Star 再计 +1

## 3.5 删除与下架
- **下架**：skill 从市场中不再展示与安装，但已安装用户不受影响
- **删除**：逻辑归档，skill 整个历史版本统一归档，不再作为活跃 skill 使用
