# 7. 页面架构与导航

## 7.1 一级入口

当前正式桌面端已经不再使用“左侧主导航 + 多个一级页面”的旧结构，而统一收敛为**顶栏分段导航**。

产品采用游客优先模式：
- `主页 / 社区 / 本地` 默认可见
- `管理` 仅在**在线且具备任一管理员菜单权限**时显示

| # | 一级入口 | 角色可见性 | 说明 |
|---|----------|-----------|------|
| 1 | 主页 | 全部 | 中心探索入口，只承载 Agent 探索主舞台 |
| 2 | 社区 | 全部 | Skill 发现、搜索、筛选、排序、安装与详情；MCP/插件仅占位 |
| 3 | 本地 | 全部 | 统一收口已安装 Skill、工具、项目与诊断相关能力 |
| 4 | 管理 | 在线管理员 | 审核、Skills、部门、用户统一工作台 |

补充说明：
- 通知固定为顶栏右上角铃铛，不作为一级页面
- 发布中心不再占据一级导航，改为主页快捷入口与社区右上角主按钮，并以覆盖层工作台承载
- 目标管理不再占据一级导航，相关能力迁入“本地”页内部的 `工具 / 项目 / 诊断`
- 设置、登录、Skill 详情、发布中心、诊断优先使用模态 / 抽屉 / 覆盖层

## 7.2 全局布局

```text
┌─────────────────────────────────────────────────────────────┐
│ 品牌区      主页 / 社区 / 本地 / 管理      通知 / 头像入口 │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                    当前一级入口舞台区                       │
│  - 主页：单舞台 Agent 探索                                  │
│  - 社区：左侧入口 + 顶部功能栏 + 搜索/卡片/热榜             │
│  - 本地：左侧入口 + 顶部功能栏 + 中间列表 + 右侧详情         │
│  - 管理：左侧入口 + 顶部功能栏 + 列表/详情联动              │
└─────────────────────────────────────────────────────────────┘
```

### 顶栏

| 元素 | 说明 |
|------|------|
| Logo / 应用名称 | 点击回主页 |
| 顶栏分段导航 | 一级切换：主页、社区、本地、管理 |
| 通知铃铛 | 显示未读数量角标；点击打开通知面板 |
| 账号入口 | 游客态显示本地模式 / 登录；登录后显示身份、角色、设置和退出 |

### 页面内部结构

- **主页**：纯 Agent 探索舞台，标题 + 提问编辑框 + 上下文 pills
- **社区**：左侧 `Skill / MCP / 插件` 入口，顶部功能栏右侧保留“发布 Skill”按钮
- **本地**：左侧 `Skills / 工具 / 项目` 入口，中间主列表，右侧当前对象详情
- **管理**：左侧 `审核 / Skills / 部门 / 用户` 入口，主区为统一列表与详情联动

### 覆盖层规则

- 登录、设置、发布中心、诊断、Skill 详情、启用范围都位于顶栏之上
- Skill 详情当前正式交付为覆盖层，而非独立页面
- 诊断从“本地”顶部操作进入，不再作为一级导航或本地页内部常驻页签

## 7.3 角色菜单差异

| 入口 / 能力 | 游客 / 普通用户 | 在线管理员 |
|-------------|:---------------:|:----------:|
| 主页 | ✅ | ✅ |
| 社区 | ✅ | ✅ |
| 本地 | ✅ | ✅ |
| 管理 | ❌ | 仅在线且具备权限时显示 |
| 通知铃铛 | ✅ | ✅ |
| 设置 | ✅ | ✅ |
| 发布中心覆盖层 | ✅（触发时先登录） | ✅ |
| Skill 详情覆盖层 | ✅ | ✅ |

管理员权限仍以后端 `menuPermissions` 为准，前端只负责将：
- `review` -> `管理 / 审核`
- `admin_skills` -> `管理 / Skills`
- `admin_departments` -> `管理 / 部门`
- `admin_users` -> `管理 / 用户`

## 7.4 信息架构

```mermaid
graph TD
    subgraph 顶栏一级导航
        HOME["主页"]
        COMMUNITY["社区"]
        LOCAL["本地"]
        MANAGE["管理（管理员在线且有权限时）"]
    end

    COMMUNITY --> C1["Skill"]
    COMMUNITY --> C2["MCP（预留）"]
    COMMUNITY --> C3["插件（预留）"]
    COMMUNITY -.-> PUBLISH["发布中心覆盖层"]

    LOCAL --> L1["Skills"]
    LOCAL --> L2["工具"]
    LOCAL --> L3["项目"]
    LOCAL -.-> DIAG["诊断覆盖层"]

    MANAGE --> M1["审核"]
    MANAGE --> M2["Skills"]
    MANAGE --> M3["部门"]
    MANAGE --> M4["用户"]

    COMMUNITY --> DETAIL["Skill 详情覆盖层"]
    LOCAL --> DETAIL
```

## 7.5 页面跳转关系

### 从主页出发

```mermaid
flowchart LR
    HOME["主页"]
    HOME -->|"提交查询"| COMMUNITY["社区 > Skill"]
    HOME -->|"通知"| NOTIFY["右上角通知面板"]
    HOME -.->|"发布"| PUBLISH["发布中心覆盖层"]
```

### 从社区出发

```mermaid
flowchart LR
    COMMUNITY["社区 > Skill"]
    COMMUNITY -->|"点击卡片"| DETAIL["Skill 详情覆盖层"]
    COMMUNITY -->|"点击安装"| INSTALL["本地安装 / 更新"]
    COMMUNITY -->|"右上角发布"| PUBLISH["发布中心覆盖层"]
```

### 从本地出发

```mermaid
flowchart LR
    LOCAL["本地"]
    LOCAL -->|"Skills"| SKILLS["本地 Skill 列表 + 右侧详情"]
    LOCAL -->|"工具"| TOOLS["工具列表 + 右侧详情"]
    LOCAL -->|"项目"| PROJECTS["项目列表 + 右侧详情"]
    LOCAL -.->|"诊断"| DIAG["诊断覆盖层"]
    SKILLS -->|"查看详情"| DETAIL["Skill 详情覆盖层"]
```
