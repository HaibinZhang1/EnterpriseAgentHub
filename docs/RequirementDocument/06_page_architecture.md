# 7. 页面架构与导航

## 7.1 一级页面

完整产品蓝图包含以下一级页面。当前 Desktop 阶段采用游客优先模式：本地页签默认可见，管理员页签只在在线且具备菜单权限时显示。

| # | 页面 | 角色可见性 | 阶段 | 说明 |
|---|------|-----------|------|------|
| 1 | 🏠 首页 | 全部 | P1 | 系统入口，P1 仅展示使用状态、推荐、最近更新、通知摘要 |
| 2 | 🛒 市场 | 全部 | P1 | Skill 发现、搜索、浏览、详情与安装 |
| 3 | 📦 我的 Skill | 全部 | P1 / P2 | P1 交付"已安装"；"我发布的 / 发布 Skill"归 P2 |
| 4 | ✅ 审核 | 在线管理员 | 当前阶段 | 最小可用只读审核页：待审核、审核中、已审核与详情查看 |
| 5 | ⚙️ 管理 | 在线管理员 | 当前阶段 | 部门/用户/Skill 管理 |
| 6 | 🔧 工具 | 全部 | P1 | 本机 AI 工具管理 |
| 7 | 📁 项目 | 全部 | P1 | 项目维度 Skill 启用管理 |
| 8 | 🔔 通知 | 全部 | P1 / P2 | P1 交付应用内消息中心；发布/审核类通知随 P2 启用 |
| 9 | ⚙️ 设置 | 全部 | P1 | 偏好设置 |
| 10 | 🔌 MCP | 预留 | P3 | P1/P2 不展示 |
| 11 | 🧩 插件 | 预留 | P3 | P1/P2 不展示 |

### 顶栏

| 元素 | 说明 |
|------|------|
| Logo / 应用名称 | 点击回首页 |
| 全局搜索栏 | 快速搜索 skill，跳转市场搜索结果 |
| 通知图标 | 显示未读通知数量角标 |
| 账号入口 | 游客态显示“本地模式 / 登录”；登录后显示身份、角色和退出 |

### 侧栏

- 固定在左侧，可收起为图标模式
- 游客默认展示：首页、市场、我的 Skill、工具、项目、通知、设置
- 登录且服务端 `menuPermissions` 包含对应项时追加展示："审核"、"管理"
- "审核"在有待审核时显示数量角标
- "通知"在有未读通知时显示角标
- "设置"置于侧栏底部，与主导航分隔

## 7.5 页面跳转关系

### 从首页出发

```mermaid
flowchart LR
    HOME["首页"]
    HOME -->|"可更新数量"| MY1["我的 > 已安装"]
    HOME -->|"热门 Skill"| M3["市场 > 详情页"]
    HOME -->|"通知摘要"| NOTIFY["通知"]
    HOME -.->|"管理员在线时"| R1["审核 > 待审核"]
    HOME -.->|"P2 被退回数量"| MY2["我的 > 我发布的"]
    HOME -.->|"P2 发布 Skill"| MY3["我的 > 发布"]
```
