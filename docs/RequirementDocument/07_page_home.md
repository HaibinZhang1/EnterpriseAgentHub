# 8.1 主页

## 页面目标
主页不再承担“后台摘要页”或“运营首页”的角色，而收敛为桌面客户端的中心探索入口。

当前正式交付的主页只回答一件事：

**从哪里开始向 Agent 发起探索。**

## 页面骨架

主页采用单舞台布局，不再保留左右工作入口、推荐区、通知摘要区或热榜区。

```text
┌──────────────────────────────────────────────────────────────┐
│                         Agent 探索                           │
│                    标题 + 多行提问编辑框                     │
│                    底部上下文 pills / 工具条                 │
└──────────────────────────────────────────────────────────────┘
```

## 核心模块

### 1. 标题区
- 标题固定为：`Agent 探索`
- 标题位于舞台上半区，不做完全垂直居中

### 2. 提问编辑框
主页输入区不是单行搜索框，而是类编辑器的多行 prompt composer：

- 多行输入区域
- placeholder：提示用户可提问、添加文件、输入命令、使用技能
- 下方工具条：权限、模型、模式、语音、发送
- 底部上下文 pills：当前工作区、模式、身份或分支等上下文

### 3. 提交行为
- 提交后将查询写入社区 Skill 搜索条件
- 自动跳转到 `社区 > Skills`

## 不再保留的内容

当前主页不再承载以下模块：
- 本机摘要指标
- 快捷入口磁贴
- 推荐 Skill
- 最近更新
- 通知摘要
- 热榜
- 发布入口大区块

以上能力改为分散到：
- 社区：发现、搜索、安装、发布入口
- 本地：已安装 Skill、工具、项目
- 顶栏通知：通知摘要与跳转
- 覆盖层：发布中心、设置、登录、Skill 详情

## 设计原则
- 主舞台必须尽量纯净
- 不做营销页
- 不叠加多块摘要卡片
- 输入体验优先于统计信息
- 主页默认服务游客优先、本地优先路径
