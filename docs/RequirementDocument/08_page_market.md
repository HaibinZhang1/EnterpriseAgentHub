# 8.2 社区（原市场）

## 页面目标
社区承担 Skill 的发现、搜索、筛选、排序、安装与详情查看能力。

当前正式交付中，“市场”页面已经按新的 IA 收口为一级入口 **社区**。

> 说明：社区页属于当前版本 Desktop 核心范围。当前版本支持搜索、排序、标签筛选、卡片网格、热榜侧栏、安装/更新入口与发布按钮；MCP / 插件仅保留占位态。

## 页面结构
- 左侧社区入口：`Skills / MCP / 插件`
- 顶部统一功能栏
- 搜索与排序区
- 标签按钮行
- Skill 卡片网格
- 右侧热榜侧栏
- 右上角 `发布 Skill` 主按钮

## 搜索能力
支持按以下字段搜索：
- skill 名称
- 描述
- 标签
- 作者
- 部门
- skillID

## 筛选能力
支持按以下维度筛选：
- 分类：固定中文短分类单选，包含 `开发 / 测试 / 文档 / 设计 / 运维 / 安全 / 集成 / 自动化 / 数据 / 知识 / 其他`
- 标签：固定中文短标签多选，默认 OR 逻辑，命中任一标签即可
- 排序
- 其他高阶筛选维度仍由服务端字段逐步增强

当前正式页面优先交付：
- 搜索词
- 分类行
- 标签多选行
- 排序

## Skill 卡片

社区当前采用卡片网格，而不是单列大摘要列表。

每张卡片展示：
- 首字母图标
- 显示名称
- 简短描述
- 分类
- 最多 3 个标签，超出显示 `+N`
- 当前状态
- 作者
- 作者部门
- 当前版本
- Star 数
- 下载量

### 卡片操作
- 主操作：安装 / 更新 / 管理范围 / 启用范围
- 次操作：详情
- 收藏 / 取消收藏
- 安装下载凭证发放并完成本地写入后，前端需刷新该 Skill 详情并即时更新下载量、热榜和详情覆盖层中的统计展示

## Skill 详情

当前正式交付的 Skill 详情优先通过覆盖层展示，而非独立详情页。

完整详情展示：

详情覆盖层采用单页阅读结构：顶部为 Skill 基础信息，操作按钮固定在覆盖层右上角；中部为概览、兼容与已启用位置；README 之前展示版本历史，底部展示 `README.md` 的完整 Markdown 内容。

### 基础信息
- skillID
- 显示名称
- 描述
- 作者
- 所属部门
- 当前版本
- 本地版本
- 更新时间
- 公开级别

### 使用与兼容信息
- 适用工具
- 支持环境说明
- 分类与标签
- 已启用位置

### 内容信息
- 版本历史：按发布时间倒序展示版本号、发布时间、风险等级、变更说明，并标记当前版本
- `README.md`：作为详情底部主阅读区，展示 Skill 包中 README.md 的完整 Markdown 内容；缺失时显示空态

### 审核与安全信息
- 风险等级
- 审核意见摘要
- 风险说明

### 操作区
操作按钮位于详情覆盖层右上角，关闭按钮保留在最右侧。

- 安装
- 更新
- 收藏 / 取消收藏
- 启用范围
- 查看已启用位置

## 受限详情
当用户属于“摘要公开但无详情权限”时，仅展示：
- skill 名称
- 发布人
- 部门
- Star 数
- 下载量
- 分类 / 图标（如有）
- “该 skill 暂未向你开放详情”提示

## 热榜侧栏

社区右侧保留独立热榜侧栏，不再回到主页。

当前版本默认展示简单热榜能力：
- 热榜
- 下载榜
- Star 榜
- 默认展示热榜
- 用户通过页签手动切换热榜、Star 榜与下载榜；榜单不自动轮播
- 榜单行点击后进入 Skill 详情覆盖层

榜单行字段：
- 首字母图标
- Skill 名称
- 单个分类标签
- 发布者名称
- 发布者部门
- Star
- 下载量

展示规则：
- 单次展示 10 条
- 三个榜单不同时上下排列，只展示当前手动选中的榜单
- 热榜按近 7 天趋势热度排序：`hotScore = (recentDownloadCount * 6 + recentStarCount * 3 + updateBoost) * riskMultiplier`
- `recentDownloadCount` 为近 7 天安装下载凭证按用户去重数；`recentStarCount` 为近 7 天新增且当前仍有效的 Star 数；当前版本更新时间在近 7 天内时 `updateBoost = 2`，否则为 `0`
- 风险倍率：低风险和未知风险为 `1.0`，中风险为 `0.8`，高风险为 `0.5`
- 热榜只展示 `hotScore > 0` 的 Skill；并列时按近 7 天安装下载用户数、近 7 天新增有效 Star、累计下载量、累计 Star、最近更新时间、名称排序
- 下载榜按安装下载用户数降序排列，Star 作为并列时的次级排序；Star 榜按 Star 数降序排列，下载量作为并列时的次级排序；仍并列时按名称排序
- 下载量不统计更新、文件预览、审核包下载或同一用户的重复安装下载凭证

## 发布入口

发布中心不再作为左侧入口或一级导航。

当前社区通过顶部功能栏右侧主按钮提供：
- `发布 Skill`

行为规则：
- 已登录：打开发布中心覆盖层
- 未登录：先打开登录模态，再进入发布流程
