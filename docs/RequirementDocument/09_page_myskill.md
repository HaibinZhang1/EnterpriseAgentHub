# 8.3 本地 Skill 与发布中心

当前设计不再使用“已安装”和“发布中心”两个一级导航，而是改为：

1. **本地** 一级入口中的 `Skills` 子视图
2. **社区** 一级入口中的作者工作区 `发布 / 我的`

> 说明：Desktop 当前已经交付完整作者治理闭环。“本地 > Skills”继续承担本机使用闭环；作者治理入口当前收口到 `社区 > 发布 / 我的`，提供真实上传、初审、审核状态、撤回、重新提交、发布新版本、上下架、归档和权限变更入口。

---

## 8.3.1 本地 > Skills

### 页面结构
- 左侧：本地入口切换（Skills / 工具 / 项目）
- 中间：Skill 搜索、状态筛选、目标筛选、本地 Skill 主列表
- 右侧：当前选中 Skill 的详情与主操作

### 展示字段
- skill 名称
- 当前本地版本
- 市场最新版本
- 是否有更新
- 已启用工具 / 项目数量
- 最近更新时间
- 启用状态
- 是否受权限收缩限制
- 异常摘要（如有）
- 从已配置 AI 工具 skills 目录和项目 skills 目录扫描到的未托管副本，若根目录包含 `SKILL.md`，需在本地 Skill 列表中可见，并标明来源位置、版本和状态
- 根目录不包含 `SKILL.md` 的 `.mdc` / `.md` / `AGENTS.md` 等目标格式文件不进入本地 Skill 主列表

### 操作
- 更新
- 启用 / 调整启用范围
- 卸载
- 查看完整详情
- 对可导入的未托管扫描项提供“纳入管理”
- 未纳入 Central Store 管理的扫描项不提供卸载与启用范围调整，只提供社区详情（如有同名 Skill）和重新扫描入口

### 目标筛选
- 支持按 `全部目标 / 工具 / 项目` 过滤本地 Skill
- 选择工具或项目后，可继续选择具体工具或具体项目
- 已安装 Skill 按启用目标匹配；从扫描发现的未托管 Skill 按来源目标匹配

### 未托管 Skill 纳入管理规则
- 纳入管理只发生在本机：复制来源目录到 Central Store，生成本地安装记录，不上传服务器，不创建发布记录
- 导入后默认认领原扫描路径为已启用目标；同一 skillID 且 checksum 一致的其他来源一并认领
- 同名但 checksum 不一致的来源保持冲突状态，不自动认领
- 若 Central Store 已有同名 Skill，弹窗提示冲突，默认推荐重命名导入；用户也可选择保留现有或确认替换本地副本
- 本地导入项显示为“本地托管”，不参与市场更新
- 后续卸载本地导入项时，确认后删除 Central Store 副本和已认领目标路径
- `SKILL.md` 中的 frontmatter 优先提供显示名称、描述和版本；版本缺失时使用 `0.0.0-local`

### 特殊规则
- 权限收缩后，如用户已安装，则保留当前本地副本
- 已安装但未启用的用户，可继续启用当前版本
- 权限收缩后按钮需明确提示“可继续使用当前版本，但不可更新”

## 8.3.2 社区 > 我的

### 展示字段
- skill 名称
- skillID
- 当前版本
- 当前状态
- 授权范围
- 公开级别
- 最近更新时间
- 下载量
- Star 数

### 操作
- 点击条目查看详情
- 撤回待审
- 重新提交
- 下架
- 上架
- 删除（归档）
- 发布新版本
- 修改权限范围 / 公开级别

### 详情增强
在“我发布的”详情区中，需突出：
- 当前审核阶段
- 审核意见
- 拒绝原因
- 历史版本
- 当前授权范围
- 当前公开级别
- 权限变更记录
- 提交包文件列表
- 文件在线预览 / 下载

### 文件预览规则
- 支持 `.md` / `.markdown` / `.txt` 在线预览
- 其他文件仅提供下载，并提示“不支持预览”
- 默认优先选中 `SKILL.md`；不存在时选第一个可预览文件
- 单文件预览内容超过 256 KB 时显示“内容已截断”

## 8.3.3 社区 > 发布

### 表单布局
- 发布页采用单列表单流，不再使用“左侧主表单 + 右侧辅助栏”的双列结构
- 上传文件与基础信息位于上方，公开与授权、分类与兼容性合并到同一列并放在基础信息下方
- 公开级别与授权范围并列展示
- 分类与标签竖向排列，标签位于分类下方
- 适用工具与适用系统横向并列展示；窄屏时可折叠为单列

### 上传方式
- 上传 skill 文件夹：必须通过文件夹选择或目录拖拽进入，不能把单个散文件当作文件夹提交
- 上传 zip 包：必须是单个 `.zip` 文件

### 表单字段
- skillID
  - 唯一标识
  - 安装后文件夹名称
  - 规则：小写字母、数字、连字符
- 显示名称
- 描述
- 分类
  - 必选 1 个
  - 固定中文短分类：开发、测试、文档、设计、运维、安全、集成、自动化、数据、知识、其他
- 标签
  - 选择 1 到 5 个
  - 固定中文短标签：代码、审查、重构、提示、规范、清单、文档、写作、测试、验收、前端、可访问、设计、运维、值班、事故、安全、权限、集成、适配、自动化、发布、数据、分析、入门、培训
  - 不支持自由输入，不进入额外审核
- 版本号
  - 默认 1.0.0
  - 需遵循 SemVer 规范
  - 版本号必须递增
- 变更说明
- 授权范围
- 公开级别
  - 非公开
  - 摘要公开
  - 详情公开
  - 全员可安装

### 上传后的系统行为
- 自动解析 skill 信息（含 SKILL.md frontmatter）
- 自动填充可识别字段
- SKILL.md frontmatter 只自动填充名称、描述等基础字段；分类和标签必须由发布者在固定中文选项中确认
- 默认不展示完整预检查清单；仅在存在未通过项时展示对应问题
- 支持用户确认与修改
- 提交成功后清空当前发布表单，并进入作者提交详情区
- 提交详情支持文件预览 / 下载
- 系统自动生成 manifest.json（数据来源于表单）

### 发布预检查项
发布工作区在提交前至少校验以下预检查项；通过项不展示，未通过项才展示问题：
- 存在 `SKILL.md`：文件夹上传和 zip 上传都必须在提交前识别到 `SKILL.md`。
- Slug 合法：skillID 必填，仅允许小写字母、数字和单个连字符，不能以连字符开头或结尾。
- Slug 必填错误默认不展示；当用户提交发布表单，或已输入但格式不合法时，再展示对应错误提示。
- Slug 不重复：首次发布不得使用已有且未归档 Skill 的 skillID；若原 Skill 已归档，允许重新走首次发布并复用原 skillID。
- SemVer 合法：版本号使用 `x.y.z`。
- 描述已填写。
- 首次发布和版本更新必须填写变更说明；权限变更可不填写代码变更说明。
- 分类已选择且属于固定中文短分类。
- 标签为 1 到 5 个固定中文短标签，且不得重复。
- 包大小不超过 5MB。
- 文件夹上传可在前端确认文件数不超过 100；zip 内部文件数由后端解压后确认。
- 脚本风险与深层结构校验当前只提示待后端或人工复核，不做客户端阻断。

只有阻塞项均通过时才允许提交发布；提交后服务端仍需执行同等或更严格校验。

### 按钮
- 提交发布申请 / 提交版本更新 / 提交权限变更
- 上一步 / 下一步
- 发布表单内容超过窗口高度时必须可纵向滚动，提交按钮在桌面和移动视口内都必须可到达

### 已发布 Skill 的权限范围修改
已发布 skill 支持对以下内容进行独立修改：
- 授权范围
- 公开级别

规则如下：
- 权限扩大：按扩大后的公开级别重新计算审核链路
- 权限收缩：简化审核，仅需同级管理员确认
- 权限修改不变更 skill 代码版本号
- 系统记录权限变更历史
- 详情区需展示当前生效权限与历史变更记录
- 权限收缩后默认遵循“存量继续使用，但禁止新增安装与更新”
- 已发布 skill 可下架；已下架 skill 可上架；已归档 skill 不可直接恢复；如需继续使用相同 slug，必须重新走首次发布审核

## 8.3.4 入口规则

作者工作区当前通过以下入口进入：
- 社区左侧作者入口 `发布`
- 通知跳转（作者审核进度）

未登录时：
- 先打开登录模态
- 登录成功后再进入社区内对应作者工作区
