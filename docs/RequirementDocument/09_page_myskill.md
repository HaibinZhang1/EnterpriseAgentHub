# 8.3 本地 Skill 与发布中心

当前设计不再使用“已安装”和“发布中心”两个一级导航，而是改为：

1. **本地** 一级入口中的 `Skills` 子视图
2. **发布中心覆盖层**

> 说明：Desktop 当前已经交付完整作者治理闭环。“本地 > Skills”继续承担本机使用闭环；“发布中心”通过覆盖层提供真实上传、初审、审核状态、撤回、重新提交、发布新版本、上下架、归档和权限变更入口。

---

## 8.3.1 本地 > Skills

### 页面结构
- 左侧：本地入口切换（Skills / 工具 / 项目）
- 中间：Skill 搜索、状态筛选、本地 Skill 主列表
- 右侧：当前选中 Skill 的详情与主操作

### 展示字段
- skill 名称
- 当前本地版本
- 市场最新版本
- 是否有更新
- 已启用工具 / 项目数量
- 最近更新时间
- 启用状态
- 是否受权限收缩限制
- 异常摘要（如有）

### 操作
- 更新
- 启用 / 调整启用范围
- 卸载
- 查看完整详情

### 特殊规则
- 权限收缩后，如用户已安装，则保留当前本地副本
- 已安装但未启用的用户，可继续启用当前版本
- 权限收缩后按钮需明确提示“可继续使用当前版本，但不可更新”

---

## 8.3.2 发布中心覆盖层 > 我发布的

### 展示字段
- skill 名称
- skillID
- 当前版本
- 当前状态
- 授权范围
- 公开级别
- 最近更新时间
- 下载量
- Star 数

### 操作
- 查看详情
- 撤回待审
- 重新提交
- 下架
- 上架
- 删除（归档）
- 发布新版本
- 修改权限范围 / 公开级别

### 详情增强
在“我发布的”详情区中，需突出：
- 当前审核阶段
- 审核意见
- 拒绝原因
- 历史版本
- 当前授权范围
- 当前公开级别
- 权限变更记录
- 提交包文件列表
- 文件在线预览 / 下载

### 文件预览规则
- 支持 `.md` / `.markdown` / `.txt` 在线预览
- 其他文件仅提供下载，并提示“不支持预览”
- 默认优先选中 `SKILL.md`；不存在时选第一个可预览文件
- 单文件预览内容超过 256 KB 时显示“内容已截断”

## 8.3.3 发布中心覆盖层 > 发布 Skill

### 上传方式
- 上传 skill 文件夹
- 上传 zip 包

### 表单字段
- skillID
  - 唯一标识
  - 安装后文件夹名称
  - 规则：小写字母、数字、连字符
- 显示名称
- 描述
- 标签 / 分类
- 版本号
  - 默认 1.0.0
  - 需遵循 SemVer 规范
  - 版本号必须递增
- 变更说明
- 授权范围
- 公开级别
  - 非公开
  - 摘要公开
  - 详情公开
  - 全员可安装

### 上传后的系统行为
- 自动解析 skill 信息（含 SKILL.md frontmatter）
- 自动填充可识别字段
- 展示上传预检查结果
- 支持用户确认与修改
- 提交成功后进入作者提交详情区
- 提交详情支持文件预览 / 下载
- 系统自动生成 manifest.json（数据来源于表单）

### 按钮
- 发布提交
- 上一步 / 下一步

### 已发布 Skill 的权限范围修改
已发布 skill 支持对以下内容进行独立修改：
- 授权范围
- 公开级别

规则如下：
- 权限扩大：按扩大后的公开级别重新计算审核链路
- 权限收缩：简化审核，仅需同级管理员确认
- 权限修改不变更 skill 代码版本号
- 系统记录权限变更历史
- 详情区需展示当前生效权限与历史变更记录
- 权限收缩后默认遵循“存量继续使用，但禁止新增安装与更新”
- 已发布 skill 可下架；已下架 skill 可上架；已归档 skill 不可恢复

## 8.3.4 入口规则

发布中心当前通过以下入口进入：
- 社区右上角 `发布 Skill` 主按钮
- 通知跳转（作者审核进度）

未登录时：
- 先打开登录模态
- 登录成功后再进入发布中心覆盖层
