# 8.3 我的 Skill

"我的 Skill"拆分为三个子页签：

1. 已安装
2. 我发布的
3. 发布 Skill

> 阶段说明：P1 Desktop 只交付"已安装"使用闭环。"我发布的"和"发布 Skill"归 P2 企业治理闭环；若 P1 需要展示占位入口，应明确显示"后续版本开放"，不进入发布流程。

---

## 8.3.1 已安装

### 展示字段
- skill 名称
- 当前本地版本
- 市场最新版本
- 是否有更新
- 已启用工具
- 已启用项目
- 最近启用时间
- 启用状态
- 是否受权限收缩限制

### 操作
- 启用 / 停用
- 更新
- 卸载
- 查看详情
- 编辑启用范围（在哪些工具 / 项目启用）

### 特殊规则
- 权限收缩后，如用户已安装，则保留当前本地副本
- 已安装但未启用的用户，可继续启用当前版本
- 权限收缩后按钮需明确提示"可继续使用当前版本，但不可更新"

---

## 8.3.2 我发布的

> 阶段：P2。该页签依赖发布、审核、权限变更和版本治理能力，不作为 P1 Desktop 交付范围。

### 展示字段
- skill 名称
- skillID
- 当前版本
- 当前状态
- 授权范围
- 公开级别
- 最近更新时间
- 下载量
- Star 数

### 操作
- 查看详情
- 撤回待审
- 重新提交
- 下架
- 上架
- 删除（归档）
- 发布新版本
- 修改权限范围 / 公开级别

### 详情增强
在"我发布的"详情页中，需突出：
- 当前审核阶段
- 审核意见
- 拒绝原因
- 历史版本
- 当前授权范围
- 当前公开级别
- 权限变更记录

---

## 8.3.3 发布 Skill

> 阶段：P2。P1 市场数据来自服务端既有数据、管理员导入、种子数据或后端预置流程，不在 Desktop 中提供发布 UX。

### 上传方式
- 上传 skill 文件夹
- 上传 zip 包

### 表单字段
- skillID
  - 唯一标识
  - 安装后文件夹名称
  - 规则：小写字母、数字、连字符
- 显示名称
- 描述
- 图标（可选）
- 标签 / 分类
- 版本号
  - 默认 1.0.0
  - 需遵循 SemVer 规范
  - 版本号必须递增
- 变更说明
- 授权范围
- 公开级别
  - 非公开
  - 摘要公开
  - 详情公开
  - 全员可安装

### 上传后的系统行为
- 自动解析 skill 信息（含 SKILL.md frontmatter）
- 自动填充可识别字段
- 展示上传预检查结果
- 支持用户确认与修改
- 支持发布前预览市场展示效果
- 系统自动生成 manifest.json（数据来源于表单）

### 按钮
- 预览
- 发布提交

### 已发布 Skill 的权限范围修改
已发布 skill 支持对以下内容进行独立修改：

- 授权范围
- 公开级别

规则如下：
- 权限扩大：按扩大后的公开级别重新计算审核链路
- 权限收缩：简化审核，仅需同级管理员确认
- 权限修改不变更 skill 代码版本号
- 系统记录权限变更历史
- 详情页需展示当前生效权限与历史变更记录
- 权限收缩后默认遵循"存量继续使用，但禁止新增安装与更新"
