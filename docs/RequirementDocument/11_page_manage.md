# 8.5 管理

"管理"模块拆分为：
1. 部门管理
2. 用户管理
3. Skill 管理

> 说明：管理页已作为在线管理员真实入口交付，所有列表与写操作均由后端再次校验菜单权限、部门范围和 `adminLevel`。

---

## 8.5.1 部门管理

### 功能
- 树状结构展示部门
- 新增下级部门
- 修改部门
- 删除下级部门

### 规则
- 上级可查询本部门及**所有后代部门**
- 可在本部门或后代部门下新增下级部门
- 一级管理员可修改自身管理根部门名称（当前为“集团”），但不可删除根部门
- 一级管理员可在根部门下新增一级部门
- 其他管理员仅可修改、删除后代部门；不可修改或删除本部门及上级部门
- 每个部门必须至少有一个管理员

### 页面要求
- 树形导航
- 右侧详情面板
- 支持展开收起
- 支持查看部门下用户数、skill 数

---

## 8.5.2 用户管理

### 列表字段
- 用户名称（对外展示，允许重名）
- 手机号（唯一登录凭证）
- 所属部门
- 角色
- 状态（正常 / 冻结）
- 已发布 skill 数
- Star 数
- 最近登录时间（预留）

### 操作
- 新增用户
- 修改用户密码
- 删除用户
- 冻结用户
- 解冻用户
- 查看用户发布的 skill
- 设置用户角色（仅可设置低于自身级别的角色）
- 冻结用户后立即使其现有会话失效

> **说明：** 系统采用自建账号体系。账号由管理员开通；新增用户时用户名称、手机号、部门和角色必填，系统统一设置固定一次性初始密码 `EAgentHub123!` 并标记为待首次改密。手机号是唯一登录凭证，格式为 1 开头的 11 位数字；用户名称用于社区、Skill 作者、审核提交人等外显场景，允许重名；内部账号 ID 不对外展示。首次使用初始密码登录时，系统只返回 15 分钟有效的改密专用令牌，不创建正常会话；用户必须设置符合弱密码校验的新密码后才能进入工作台，新密码至少 12 位且同时包含大写字母、小写字母、数字和特殊字符，并且不能与初始密码相同。系统初始一个一级管理员，后续用户由上一级管理员创建和分配角色。管理端可直接修改管辖范围内用户密码；管理端修改密码会立即使该用户现有会话失效并作为最终密码生效，不触发下次登录强制改密。接入企业系统或统一身份源、批量导入用户为后续功能。

## 8.5.3 Skill 管理

### 列表字段
- skill 名称
- 发布者
- 发布部门
- 当前版本
- 当前状态
- Star 数
- 下载量
- 是否上架
- 最近更新时间

### 查询能力
- 按用户搜索
- 按 Star 数排序
- 按下载量排序
- 按状态筛选
- 按部门筛选

### 操作
- 删除（归档）
- 下架
- 上架
- 查看详情

> **说明：** 管理员只能操作本管辖范围内（本部门及所有后代部门）的 skill。跨部门下架全员可安装 skill 仅**一级管理员**可操作。
