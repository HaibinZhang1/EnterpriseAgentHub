# 8.6 工具视图

## 页面目标
在“本地”一级入口内，统一管理本机已安装 AI 工具与 skill 的绑定关系。

> 说明：当前设计已将工具、项目和诊断统一收敛到“本地”入口内部；工具视图承担已安装 Skill 的本机启用/停用和路径配置，批量同步、复杂策略下发暂不展开。

## 页面结构
- 左侧：本地入口切换（Skills / 工具 / 项目）
- 顶部：统一功能栏 + `扫描 / 诊断 / 添加工具`
- 中间：工具列表
- 右侧：当前选中工具详情

## 页面内容
- 工具列表
- 工具配置状态
- 默认路径识别结果
- 自定义路径配置
- skill 安装路径配置
- 当前工具启用状态

## 按钮
- 刷新检测工具
- 添加自定义工具

## 列表字段
- 工具名称
- 配置路径
- skills 安装路径
- 启用状态
- 检测状态
- 已启用 skill 数量

## 工具检测方式
系统通过以下方式检测本机已安装的 AI 工具：
- 注册表扫描
- 路径探测

当前内置 Codex、Claude、Cursor、Windsurf、opencode 和自定义目录，启用时采用 symlink 优先、失败自动降级 copy，并对内置工具执行真实格式转换。

## 工具详情
需展示：
- 该工具下已启用哪些 skill
- 每个 skill 的版本
- 启用状态
- 安装路径
- 配置异常提示

## 操作
- 修改配置路径
- 修改 skills 安装路径
- 开启 / 关闭当前工具配置
- 对单个 skill 启用 / 停用
- 重新扫描

# 8.7 项目视图

## 页面目标
允许用户在“本地”入口内手动维护项目，并在项目维度配置 skill。

> 说明：当前设计已将工具、项目和诊断统一收敛到“本地”入口；项目视图当前支持手动项目与项目级启用，公共 Skill 继承、禁用继承、批量目标管理暂不展开。

## 页面结构
- 左侧：本地入口切换
- 顶部：统一功能栏 + `扫描 / 诊断 / 添加项目`
- 中间：项目列表
- 右侧：当前选中项目详情

## 功能
- 添加项目
- 设置项目 skills 安装路径
- 项目启用状态管理
- 项目级启用 skill

## 列表字段
- 项目名称
- 项目路径
- skills 安装路径
- 启用状态
- 已启用 skill 数量

## 项目详情
展示：
- 已额外启用的项目 skill
- 项目最终生效的 skill 列表预览
- 项目路径与更新时间

## 项目优先级规则
- 项目级启用优先于工具级启用
- 项目路径优先于工具路径
- 当同一 skill 在项目与工具中配置冲突时，以项目内最终落地的版本和路径为准

# 8.8 诊断覆盖层

## 页面目标
集中展示工具与项目目标的扫描摘要、异常发现和跳转入口。

## 页面内容
- 目标扫描摘要
- unmanaged / conflict / orphan 聚合发现
- 从异常项跳转回对应工具或项目目标

## 入口规则
- 从“本地”顶部功能栏中的 `诊断` 进入
- 不再作为一级导航或本地页内部常驻页签
