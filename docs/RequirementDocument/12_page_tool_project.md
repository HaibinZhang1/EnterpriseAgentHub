# 8.6 工具

## 页面目标
统一管理本机已安装 AI 工具与 skill 的绑定关系。

> 阶段说明：工具页属于 P1 Desktop 核心范围，优先支撑已安装 Skill 的本机启用/停用和路径配置；批量同步、复杂策略下发进入 P3。

## 页面内容
- 工具列表
- 工具配置状态
- 默认路径识别结果
- 自定义路径配置
- skill 安装路径配置
- 当前工具启用状态

## 按钮
- 刷新检测工具
- 添加自定义工具

## 列表字段
- 工具名称
- 配置路径
- skills 安装路径
- 启用状态
- 检测状态
- 已启用 skill 数量

## 工具检测方式
系统通过以下方式检测本机已安装的 AI 工具：
- **注册表扫描**：读取 Windows 注册表中的安装信息
- **路径探测**：检查预定义的默认安装路径

> 参考 [P1 Tool Adapter 配置契约](22_p1_tool_adapter_contract.md) 中的 Tool Adapter 配置。P1 内置 Codex、Claude、Cursor、Windsurf、opencode 和自定义目录，启用时仅使用 copy，并对内置工具执行真实格式转换。

## 工具详情
需展示：
- 该工具下已启用哪些 skill
- 每个 skill 的版本
- 启用状态
- 安装路径
- 配置异常提示

## 操作
- 修改配置路径
- 修改 skills 安装路径
- 开启 / 关闭当前工具配置
- 对单个 skill 启用 / 停用
- 一键同步 / 重新扫描（预留）

## 启用与路径冲突规则
为避免工具级、项目级重复配置导致歧义，统一采用以下规则：

- 同一个 skill 不允许多版本并存（每台设备本地只能有一个版本）
- 工具级启用和项目级启用冲突时，**项目级优先**
- 工具路径与项目路径同时存在时，**项目路径优先**
- 若启用动作会覆盖现有文件，系统直接覆盖，不提供"保留本地修改再启用"
- 启用动作仅通过 copy 分发转换后的产物，不使用 symlink
- 卸载时先提示当前被哪些工具 / 项目引用，用户确认后直接卸载
- 卸载动作不做引用校验阻断，由用户二次确认承担结果

# 8.7 项目

## 页面目标
允许用户手动维护项目，并在项目维度配置 skill。

> 阶段说明：项目页属于 P1 Desktop 核心范围，P1 仅支持手动项目与项目级启用。公共 Skill 继承、禁用继承、批量目标管理进入 P3。

## 功能
- 添加项目
- 设置项目 skills 安装路径
- 项目启用状态管理
- 项目级启用 skill

## 列表字段
- 项目名称
- 项目路径
- skills 安装路径
- 启用状态
- 已启用 skill 数量

## 项目详情
展示：
- 已继承的公共 skill（预留）
- 已额外启用的项目 skill
- 已禁用的公共 skill（预留）
- 项目最终生效的 skill 列表预览

## 项目优先级规则
- 项目级启用优先于工具级启用
- 项目路径优先于工具路径
- 当同一 skill 在项目与工具中配置冲突时，以项目内最终落地的版本和路径为准
