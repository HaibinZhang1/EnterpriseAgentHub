# 10. 核心流程说明

> 说明：本页同时记录发布治理流程与 Desktop 本机安装/启用流程；当前仓库已经包含真实作者、审核、管理与本机闭环实现。

## 10.1 Skill 发布流程
1. 用户进入"发布 Skill"
2. 上传文件夹或 zip
3. 系统解析 skill 元信息与包结构
4. 用户补充 / 修改字段
5. 提交发布
6. skill 上传到服务器（通过服务端中转）
7. 系统自动生成 manifest.json
8. 进入"系统初审中"
9. 初审通过则进入"待管理员审核"
10. 初审发现异常则进入"待人工复核"
11. 对应管理员审核（审核链路按实时组织链路计算）
12. 审核同意：
    - 状态变为"已发布"
    - skill 进入市场
13. 审核拒绝：
    - 状态变为"审核拒绝"
    - 用户可查看拒绝原因，可重新提交
14. 退回修改：
    - 状态变为"退回修改"
    - 用户可修改后重新提交

## 10.2 Skill 版本更新流程
与首次发布流程一致（统一状态机，通过 `publish_type: update` 字段标记），但：
- 审核链路按实时组织链路重新计算
- 已发布旧版本在新版本审核期间保持可用
- 版本号必须递增（SemVer）

## 10.3 权限变更流程
- 权限范围 / 公开级别修改走独立的权限变更状态流
- 权限扩大：按扩大后的公开级别重新计算审核链路
- 权限收缩：简化审核，仅需同级管理员确认
- 权限变更不修改代码版本号，但产生权限变更记录
- 审核期间原权限配置继续生效

## 10.4 撤回流程
- 发布者可对以下状态的 skill 执行撤回：上传成功、系统初审中（终止扫描）、待人工复核、待管理员审核
- **审核中（已锁单）不可撤回**
- 撤回后状态变为"已撤回"

## 10.5 下架流程
- 已发布 skill 可被发布者或管理员下架
- 下架后市场不再展示 / 安装
- 已安装用户不受影响
- 跨部门紧急下架仅一级管理员可操作

## 10.6 上架流程
- 已下架 skill 可被发布者或管理员重新上架
- 上架后重新进入市场并恢复可安装资格
- 已归档 skill 不可重新上架
- 上架动作需保留状态变更记录

## 10.7 删除（归档）流程
- 发布者或管理员可执行删除（归档）
- 删除后整个 skill 历史版本统一归档
- 市场不再展示
- 不再作为活跃 skill 使用

## 10.8 审核 / 提交详情文件预览流程
1. 作者或审核员进入详情页
2. 服务端返回当前提交包文件列表
3. 页面默认选中 `SKILL.md`，没有则选中第一个可预览文件
4. 若文件为 `.md` / `.markdown` / `.txt`，服务端返回文本内容并在页面中只读展示
5. 若文件不支持预览，则仅提供下载入口
6. 若文件内容超过 256 KB，服务端截断并返回截断标记

## 10.9 安装与启用流程
1. 用户在市场中安装 skill
2. skill 通过**服务端中转**下载到本地 Central Store
3. 状态为"已安装未启用"
4. 用户选择工具 / 项目启用
5. 系统通过 Tool Adapter 做目标工具格式转换，并以 symlink 优先、失败自动降级 copy 的方式分发到对应工具/项目目录
6. skill 在对应工具 / 项目中生效
7. 市场与"已安装"页实时显示"已安装 / 已启用"
8. 如工具级与项目级冲突，则项目级优先
9. 如工具路径与项目路径冲突，则项目路径优先

## 10.10 卸载流程
1. 用户点击卸载
2. 系统提示当前被哪些工具 / 项目引用
3. 用户确认后直接卸载（从 Central Store 及所有目标目录移除）
4. 卸载后当前 skill 在本机不可继续使用
5. 已卸载用户后续若权限已收缩，则不能重新安装
