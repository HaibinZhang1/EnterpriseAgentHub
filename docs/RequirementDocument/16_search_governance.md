# 11. 搜索、排序与治理指标规则

## 11.1 搜索默认规则
默认搜索范围包含当前用户**可查看详情的 skill** 和 **摘要公开的 skill**。

> **V1.2 变更：** 搜索默认范围从"仅可查看详情"扩大为"可查看详情 + 摘要公开"，避免信息茧房，帮助新用户发现更多公开 skill。

## 11.2 扩展搜索规则
用户可通过筛选控制搜索范围：
- 仅显示我有权限查看的 skill
- 包含详情公开 / 全员可安装 skill
- 按固定中文分类单选筛选
- 按固定中文标签多选筛选；多个标签默认 OR 逻辑，命中任一标签即可

## 11.2.1 分类与标签规则
分类用于社区主筛选，每个 Skill 必选 1 个。当前固定分类为：
`开发 / 测试 / 文档 / 设计 / 运维 / 安全 / 集成 / 自动化 / 数据 / 知识 / 其他`。

标签用于搜索、卡片展示和组合筛选，每个 Skill 选择 1 到 5 个。当前固定标签为：
`代码 / 审查 / 重构 / 提示 / 规范 / 清单 / 文档 / 写作 / 测试 / 验收 / 前端 / 可访问 / 设计 / 运维 / 值班 / 事故 / 安全 / 权限 / 集成 / 适配 / 自动化 / 发布 / 数据 / 分析 / 入门 / 培训`。

标签不进入独立审核流程，不提供自由新增入口。后续如需新增词汇，由产品和开发直接扩充固定列表与测试种子数据。

## 11.3 搜索结果展示规则
- 完整可见：正常卡片
- 摘要可见：受限卡片，标明"仅摘要可见"
- 完全不可见：不出现在搜索结果中

---

## 11.4 下载量口径
下载量采用**按用户去重的安装下载口径**。当前版本仅在已登录用户为安装目的申请 Skill 包下载凭证时计入；同一用户、同一 Skill、同一目的只计一次。

补充规则：
- 更新包下载不计入下载量
- 审核包下载、文件预览、发布包下载、客户端更新包下载不计入 Skill 下载量
- 本地安装最终失败不会回退已经发放的安装下载凭证计数
- 下载量是去重累计值，不随卸载回退

## 11.5 Star 口径
- Star 显示为实时数量
- 用户取消 Star 后立即减 1
- 用户再次 Star 后再加 1

## 11.6 综合排序规则
综合排序至少考虑以下因子：
- 搜索相关度
- Star 数
- 下载量
- 最近更新时间
- 安全等级 / 风险等级

其中：
- 风险等级较高的 skill 在综合排序中可降权
- 降权不影响精确搜索结果命中，仅影响默认推荐顺序

## 11.7 全站曝光榜规则
- 仅作为系统给出的排行榜
- 社区侧栏当前展示热榜、Star 榜与下载榜三个榜单，每次展示前 10
- 默认展示热榜；用户可通过页签手动切换榜单，榜单不自动轮播
- 是否进入榜单由公开级别和系统排序共同决定
- 非公开 skill 不展示在全站曝光榜中
- 热榜采用服务端当前时间向前滚动 7 天的趋势热度，只展示 `hotScore > 0` 的 skill
- `recentDownloadCount` 为近 7 天安装下载凭证按用户去重数；`recentStarCount` 为近 7 天新增且当前仍有效的 Star 数；当前版本更新时间在近 7 天内时 `updateBoost = 2`，否则为 `0`
- `hotScore = (recentDownloadCount * 6 + recentStarCount * 3 + updateBoost) * riskMultiplier`
- 风险倍率：低风险和未知风险为 `1.0`，中风险为 `0.8`，高风险为 `0.5`
- 热榜按热度分、近 7 天安装下载用户数、近 7 天新增有效 Star、累计下载量、累计 Star、最近更新时间、名称排序
- 下载榜按下载量、Star、名称排序；Star 榜按 Star、下载量、名称排序
