# 19. 视觉设计系统与页面统一规范

## 页面目标
为新增页面、模块和状态视图提供统一的设计语言、布局规则、组件规格与动效约束，使 AI 或前端实现时可以直接参照本规范，保持页面风格一致、密度一致、交互节奏一致。

> 说明：本规范是当前版本 Desktop 工作台的页面视觉基线。其设计来源综合自 `AuraForge`、`Material Redesign`、`Layout Prototype` 与 `Desktop Design Prototype` 四组原型。新增页面默认采用“浅色企业 AI 工作台”风格；深色方案仅用于管理/诊断/沉浸式工作区等少数场景，不作为默认页面底色。

---

## 19.1 设计定位

### 19.1.1 统一风格定义
当前产品的统一视觉方向定义为：

**浅色企业 AI 工作台 + 克制微动效 + 桌面级高密度操作区**

### 19.1.2 风格关键词
- 专业、可信、轻量、克制
- AI 原生，但不做科幻炫技
- 信息密度高，但层级必须清楚
- 默认桌面端优先，支持窄宽度收缩
- 数据、状态、目标、结果优先于装饰性内容

### 19.1.3 新页面必须满足的感知结果
- 第一眼能看出是同一产品，而不是另一套站点
- 用户在 3 秒内能定位页面主任务、主操作、主状态
- 卡片、按钮、标签、输入框在不同页面上保持同一语法
- 页面即使模块变多，也不能退化成“堆卡片”

### 19.1.4 明确禁止
- 随机换主题色、随机换圆角体系、随机换阴影体系
- 为了“更像 AI”加入大量光效、粒子、发光描边、过度渐变
- 页面级营销 Hero 抢占工作区，影响操作密度
- 单页同时出现多种不兼容风格，例如企业后台 + 霓虹赛博 + Material 夸张色块
- 用插画替代系统状态、数据和结构表达

---

## 19.2 主题策略

### 19.2.1 默认主题
- 默认使用浅色主题
- 页面底色保持冷白、浅灰、雾面蓝灰体系
- 强调“清洁桌面工作区”，而不是“深色代码编辑器”

### 19.2.2 深色主题使用边界
仅以下场景允许采用深色主背景：
- 管理/诊断/审计类高密度工作台
- 需要突出矩阵、Inspector、风险或系统约束的页面
- 全屏沉浸式流程编排画布
- 预览区、终端区、日志区、差异对比区

### 19.2.3 同页混用规则
- 同一页面允许有 1 个深色强调区块，例如 featured banner、日志面板或沉浸工作区
- 除该强调区块外，其余卡片仍保持浅色体系
- 不允许整页出现多个互相竞争的深色强调区

| 状态 | 文字 | 背景 | 边框 | 用途 |
|------|------|------|------|------|
| Success | `#047857` | `#ECFDF5` | `#A7F3D0` | 成功、已安装、已启用、已同步 |

| Neutral | `#475569` | `#F8FAFC` | `#E2E8F0` | 中性状态、默认筛选、未启用 |

### 19.3.3 渐变规则
- 标准品牌渐变：`linear-gradient(135deg, #4F46E5, #0EA5E9, #14B8A6)`
- 品牌渐变只允许用于：
  - Hero 标题高亮词
  - 品牌徽标/品牌按钮 hover 背层
  - Featured Banner 局部装饰
  - 工作流连线或重点状态引导
- 不允许整页铺满大面积高饱和渐变底

### 19.3.4 深色保留 Token

| Token | 值 | 用途 |
|------|------|------|
| `dark-bg` | `#0A0A0A` | 深色页面主背景 |
| `dark-elevated` | `rgba(17,19,24,0.82)` | 深色卡片/浮层 |
| `dark-panel` | `rgba(255,255,255,0.05)` | 深色弱面板 |
| `dark-text` | `#F7F8FB` | 深色主题主文字 |
| `dark-text-muted` | `rgba(247,248,251,0.58)` | 深色主题次级文案 |

## 19.4 字体与排版

### 19.4.1 字体族
- UI 正文字体：`Inter`
- 展示型标题字体：`Space Grotesk`
- 数值、版本、路径、Hash、状态码：等宽字体

### 19.4.2 排版等级

| 等级 | 字号/行高 | 字重 | 用途 |
|------|-----------|------|------|
| Display XL | `40 / 48` | 800 | 页面主 Hero，仅首页或核心工作台允许 |
| H1 | `32 / 40` | 800 | 页面主标题 |
| H2 | `24 / 32` | 700 | 区块标题 |
| H3 | `20 / 28` | 700 | 卡片组标题、详情模块标题 |
| H4 | `16 / 24` | 700 | 卡片标题、行标题 |
| Body M | `14 / 22` | 500 | 标准正文 |
| Body S | `13 / 20` | 500 | 次级正文 |
| Meta | `12 / 18` | 500 | 描述、注释、辅助说明 |
| Label | `11 / 16` | 700 | Section Label、Badge、统计标签 |
| Micro | `10 / 14` | 700 | 状态小标、角标、按钮次级说明 |

### 19.4.3 文案约束
- 中文产品文案优先简洁、短句、结果导向
- 页面标题描述主任务，不描述页面本身
- 列表项描述尽量控制在 1 到 2 行
- Meta 文案不堆砌名词，优先“对象 + 状态 + 动作”

### 19.4.4 标题使用规则
- `Space Grotesk` 仅用于页面主标题、Hero、指标大数值、少量展示型模块
- 常规后台列表、表格、详情说明统一使用 `Inter`
- 一个页面最多 1 个 Display 级标题，避免争抢视觉中心

## 19.5 间距、圆角与阴影

### 19.5.1 间距体系
采用 8pt 网格，允许以下标准值：
- `4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64`

### 19.5.2 典型间距

| 场景 | 取值 |
|------|------|
| 页面左右边距 | `24`，宽屏可增至 `32` |
| 顶栏高度 | `72` 到 `80` |
| 侧栏宽度 | `240` 到 `256` |
| 页面主区块间距 | `24` 或 `32` |
| 卡片内边距 | `20` 或 `24` |
| 大型 Hero / Featured 面板内边距 | `32` 到 `40` |
| 图标与标题间距 | `10` 到 `12` |
| 按钮组间距 | `8` 到 `12` |

### 19.5.3 圆角梯度

| 名称 | 值 | 用途 |
|------|------|------|
| `radius-xs` | `6` | 小徽标、小标签 |
| `radius-sm` | `8` | 小按钮、小输入框、小卡片 |
| `radius-md` | `12` | 输入框、按钮、列表行、标准控件 |
| `radius-lg` | `16` | 节点卡、模块块 |
| `radius-xl` | `20` | 常规业务卡片 |
| `radius-2xl` | `24` | Marketplace 卡片、大块信息卡 |
| `radius-3xl` | `32` | Hero、上传区、大型容器 |
| `radius-pill` | `999` | 分段切换、Chip、Badge |

### 19.5.4 阴影层级

| 层级 | 值 | 用途 |
|------|------|------|
| `shadow-xs` | `0 2px 10px rgba(15,23,42,0.02)` | 静态轻卡片 |
| `shadow-sm` | `0 4px 12px rgba(15,23,42,0.03)` | 标准面板 |
| `shadow-md` | `0 10px 24px rgba(15,23,42,0.08)` | hover、浮层、选中块 |
| `shadow-lg` | `0 20px 40px -10px rgba(15,23,42,0.08)` | 强交互卡、Hero 悬浮态 |
| `shadow-dark` | `0 24px 64px rgba(0,0,0,0.38)` | 深色诊断页、强调 Banner |

### 19.5.5 边框规则
- 优先细边框，而不是重阴影
- 标准边框透明度控制在 `6%` 到 `12%`
- 选中态以“边框提亮 + 轻微阴影抬升”为主，不做粗边框硬切换

## 19.6 页面骨架与布局模板

### 19.6.1 全局骨架
新增页面必须优先落入以下统一骨架：

```text
顶栏（72~80）
├─ 左：品牌/返回/面包屑
├─ 中：全局搜索或页签导航
└─ 右：通知/筛选/主操作/账号

主体区
├─ 左侧一级导航（240~256，可收起）
└─ 主内容区（24~32 内边距）
```

### 19.6.2 主内容区的四种标准模板

#### A. 概览页
适用于：首页、总览、系统入口
- 顶部标题区
- 1 个主 Hero 或系统摘要卡
- 2 到 4 个指标卡
- 1 到 2 个主任务模块
- 1 个次级活动流/日志区

#### B. 工作台页
适用于：已安装、工具与项目、编排、诊断
- 左列：对象列表
- 中列：矩阵/主操作区
- 右列：Inspector / 结果 / 异常说明
- 三列需要在同屏内互相联动

#### C. 市场页
适用于：市场、插件、模板
- 标题区
- 筛选/搜索/排序条
- 1 个精选模块（可选）
- 卡片网格区
- 列表底部支持空状态与分页/延迟加载

#### D. 管理页
适用于：审核、部门、用户、Skill 管理
- 左侧二级导航或分组入口
- 中列列表
- 右列详情或操作面板
- 风格允许更深、更密，但结构不变

### 19.6.3 页面新增时的模板选择原则
- 以“主任务”而不是“页面名”决定模板
- 不能确定时优先选 `B. 工作台页`
- 只有真正需要“发现与比较”的场景才用 `C. 市场页`

## 19.7 背景、装饰与视觉氛围

### 19.7.1 页面背景
- 主背景默认使用 `canvas`
- 可叠加 1 层浅网格背景
- 网格尺寸仅允许 `32px` 或 `48px`
- 网格线透明度控制在 `0.03` 左右

### 19.7.2 环境光
- 可使用 `blue / purple / teal` 的模糊环境光块
- 光斑尺寸建议 `600` 到 `800`
- 单页最多 3 处
- 透明度 `0.2` 到 `0.6`
- 不允许出现高饱和霓虹光晕

### 19.7.3 玻璃效果
- `backdrop-filter: blur(...)` 只用于：
  - 顶栏
  - 侧栏
  - Hero / Featured 容器
  - 浮层 / 菜单 / 弹窗
- 普通业务卡片仍以实体白卡为主

### 19.7.4 渐变与插图
- 渐变是点缀，不是底色主角
- 页面若有一处深色 Banner，则其他区域必须回归克制
- 不允许用插图替代结构布局

## 19.8 图标系统

### 19.8.1 图标来源
- 默认使用 `Lucide` 风格线性图标
- 品牌型对象可使用品牌色容器，但图形仍保持简洁

### 19.8.2 图标尺寸

| 场景 | 尺寸 |
|------|------|
| 输入框、按钮内图标 | `16` |
| 顶栏、导航、行操作 | `18` |
| 卡片头部主图标 | `20` 到 `24` |
| 空状态/上传区图标 | `32` 到 `40` |
| Hero / Featured 视觉图标 | `48` 到 `80` |

### 19.8.3 图标容器规格

| 场景 | 容器尺寸 | 圆角 |
|------|-----------|------|
| 小按钮内图标容器 | `28` 到 `32` | `8` |
| 标准卡片图标容器 | `40` 到 `48` | `12` 到 `16` |
| 大卡片图标容器 | `56` 到 `64` | `16` 到 `20` |

### 19.8.4 图标颜色规则
- 默认图标使用中性灰或页面主色
- 每张卡片最多 1 个主色图标容器
- Marketplace / Plugin 卡可以使用品牌色
- 状态图标优先用 dot、check、alert、loader，不滥用装饰图标

### 19.8.5 禁止事项
- 不混用线性图标和拟物图标
- 不使用厚重 3D icon pack
- 不为普通业务卡片添加无意义大插画

## 19.9 卡片系统

### 19.9.1 指标卡
- 圆角：`20`
- 内边距：`20` 到 `24`
- 结构：Label / Value / Secondary Hint
- 数值推荐使用等宽字体
- 适用于：总数、延迟、容量、活跃状态

### 19.9.2 实体卡
- 圆角：`20` 或 `24`
- 结构：图标区 / 标题描述 / 状态 / Footer Meta
- hover 时允许 `translateY(-2 ~ -4px)`，最大 `scale(1.01)`
- hover 时必须叠加轻微变色，推荐使用主色 `4%` 到 `8%` 的浅染色背景，或在白卡上做非常轻的冷色提亮
- 适用于：Skill、插件、Agent、模板、部门、工具

### 19.9.3 列表行卡
- 行高：`64` 到 `84`
- 圆角：`12` 到 `16`
- hover 只做底色变化或轻微提亮
- 列表行 hover 不做明显位移，优先使用主色 `3%` 到 `6%` 的浅染色背景
- 适用于：日志、通知、上下文注入记录、审核列表

### 19.9.4 上传/空状态卡
- 圆角：`32`
- 虚线边框：`2px`
- 图标置中
- 说明文案控制在 2 行内
- 适用于：拖拽上传、空列表、空结果

### 19.9.5 Inspector 卡
- 圆角：`20` 到 `24`
- 分块展示“当前选中 / 最终结果 / 异常说明 / 主动作”
- 底部主动作固定在视觉闭环位置
- 适用于：部署、审核详情、工具冲突、权限说明

### 19.9.6 Featured Banner
- 每页最多 1 个
- 圆角：`32`
- 内边距：`32` 到 `40`
- 可以采用深色渐变底
- 右侧允许出现大图标或抽象图形
- 不承载复杂表单与多行操作

## 19.10 控件与组件规格

### 19.10.1 按钮

| 类型 | 高度 | 圆角 | 说明 |
|------|------|------|------|
| Primary | `40` | `12` | 深色底或主色底，承载主操作 |
| Secondary | `40` | `12` | 白底边框，承载次操作 |
| Large Primary | `44` 到 `48` | `12` | Hero、Banner 主操作 |
| Icon Button | `36` 到 `40` | `10` 到 `12` | 过滤、切换、设置 |

规则：
- 一个操作区只允许 1 个明确主按钮
- 同一行按钮不超过 3 个
- 主按钮文案优先动词开头，如“安装”“发布 Skill”“创建流程”
- 按钮 hover 默认同时包含轻微位移和轻微变色；Primary 按钮可略微加深底色，Secondary / Ghost 按钮使用主色 `4%` 到 `8%` 的浅染色背景

### 19.10.2 输入框
- 高度：`40` 到 `44`
- 圆角：`12`
- 左侧允许搜索图标
- focus ring 统一为主色 `4px` 轻透明外发光
- placeholder 使用 `text-soft`

### 19.10.3 分段导航 / Tabs / Chips
- 高度：`32` 到 `40`
- 圆角：`999`
- 选中态优先深色底或实体浅色底
- 未选中态保持弱背景 + 细边框
- hover 时允许轻微变色，推荐使用主色 `4%` 到 `6%` 的浅染色背景，不做明显跳变
- 同一分组内标签数建议不超过 8 个

### 19.10.4 Badge / 状态标签
- 高度：`20` 到 `24`
- 字号：`10` 到 `11`
- 字重：700
- 推荐大写或短词，不超过 14 个字
- 同一卡片状态标签不超过 2 个

### 19.10.5 Progress / Metric 条
- 进度条高 `6`
- 轨道使用浅灰
- 进度使用主渐变或单色
- 数值说明置于条上方右侧，不放在条中间

## 19.11 状态、反馈与空错场景

### 19.11.1 必须统一处理的状态
- 默认
- hover
- active
- selected
- disabled
- loading
- success
- warning
- danger
- empty
- offline

### 19.11.2 状态表现原则
- 成功：绿色标签或绿色 dot，避免大面积绿色底
- 风险：橙色或黄色信息条，必须附带原因
- 错误：红色仅用于真实失败、危险或不可恢复动作
- 加载：优先 spinner、skeleton、局部状态替换，不要整页闪烁
- hover：在原有轻微浮起或提亮基础上，必须补充轻微变色；浅色主题默认使用主色 `3%` 到 `8%` 的染色背景，避免只靠阴影表达可点击性

### 19.11.3 空状态文案结构
- 第一行：说明“当前没有什么”
- 第二行：说明“为什么”或“下一步能做什么”
- 允许带 1 个主操作按钮

## 19.12 动效规范

### 19.12.1 动效原则
- 动效服务于状态切换、可点击感和层级变化
- 不服务于单纯炫技
- 动效不能干扰阅读和操作效率

### 19.12.2 时间曲线

| 场景 | 时长 | 曲线 |
|------|------|------|
| Hover | `180ms` | `ease` |
| 控件切换 | `180ms` | `ease` |
| 卡片浮起 | `240ms` | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 页面切换 | `320` 到 `400ms` | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 侧栏开合 | `300ms` | `ease-in-out` |
| 骨架/脉冲 | `1.6` 到 `2s` | `ease-in-out` |
| Spinner | `1s` | `linear infinite` |

### 19.12.3 允许的动效形式
- `opacity`
- `translateY(-1 ~ -4px)`
- `scale(1.00 ~ 1.01)`
- `border-color`
- `shadow`
- `background-color`
- 状态点 pulse
- loader spin
- 轻量 fade up

### 19.12.4 标准动效模板
- 页面视图切换：`fadeUp(12px -> 0, 400ms)`
- 卡片 hover：`translateY(-2px)`；重要实体卡最多 `-4px`，并同时叠加主色 `4%` 到 `8%` 的轻微染色背景
- 侧栏开关：`translateX`
- 状态同步：旋转 loader
- 在线状态：小圆点 `pulse`
- Memory / Live 节点：轻度 `ping`

### 19.12.5 明确禁止
- 大幅缩放
- 弹簧式弹跳
- 高频闪烁
- 持续漂浮、持续抖动
- 页面级自动轮播和大面积自动位移

### 19.12.6 减弱动效要求
- 需要支持减少动效模式
- 降级后仅保留 opacity、颜色与状态切换
- 长循环动画应能关闭

## 19.14 新增页面的 AI 设计约束

### 19.14.1 AI 必须先回答的 6 个问题
在设计新页面前，AI 必须先明确：
1. 这个页面属于概览页、工作台页、市场页还是管理页？
2. 当前页面的主任务是什么？
3. 页面里的主操作是什么？
4. 页面里最重要的 1 个状态是什么？
5. 哪些信息必须同屏出现？
6. 空、错、加载、离线时页面如何收口？

### 19.14.2 AI 输出新页面时的固定结构
AI 在产出新页面设计时，必须按以下结构输出：
- 页面目标
- 页面模板类型
- 布局结构
- 组件清单
- 关键状态
- 动效说明
- 视觉 token 使用说明
- 与现有页面的继承关系

### 19.14.3 AI 不得做的事
- 不得另起一套配色
- 不得新增无来源的圆角、阴影和字体体系
- 不得把页面做成营销落地页
- 不得把所有内容包进多个嵌套卡片
- 不得为了“高级感”牺牲信息密度和主任务清晰度

## 19.15 新页面验收清单

新增页面在评审时，至少检查以下内容：

### 19.15.1 风格一致性
- 是否使用本规范中的主色、状态色和背景体系
- 是否使用本规范中的圆角梯度
- 是否与现有页面共享同一 Header / Sidebar 语法

### 19.15.2 结构一致性
- 是否已归类到四种标准模板之一
- 是否有明确主操作
- 是否存在结构重复或无意义装饰区

### 19.15.3 组件一致性
- 按钮、输入框、标签、卡片是否使用统一尺寸
- 图标尺寸与图标容器是否符合规则
- 状态标签是否采用统一色板

### 19.15.4 动效一致性
- 动效时长是否落在规范范围内
- 是否存在不必要的循环动效
- hover 是否控制在轻微浮起和提亮范围内

### 19.15.5 状态完整性
- 是否有空状态
- 是否有错误状态
- 是否有加载状态
- 是否有权限限制或离线态说明
