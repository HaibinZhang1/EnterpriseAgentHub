## 2. 产品定位

P1 只交付 Desktop 端 Skill 使用闭环，不交付发布、审核和管理台。用户应能在企业内网环境中登录客户端，确认服务连接状态，浏览市场，安装或更新 Skill，并将已安装 Skill 启用到本机工具或项目。

P1 市场数据由服务端既有数据、管理员导入、种子数据或后端预置流程提供。Desktop 客户端只消费这些数据，不提供发布 Skill、审核 Skill、维护部门或管理用户的 UX。

P1 账号采用自建账号体系，由管理员开通用户账号；后续版本再考虑接入企业系统或统一身份源。

## 3. 目标与非目标

### 3.1 目标

- 完成登录、身份展示和服务端连接状态反馈
- 完成市场浏览、搜索、筛选、排序、详情和受限详情
- 完成 Skill 安装、更新、卸载，以及本地安装状态展示
- 完成 Skill 启用/停用到工具和项目
- 完成工具路径、项目路径的基础配置和异常提示
- 完成应用内通知、基础设置和离线本地使用
- 固化 P1 所需的最小 Skill 元数据和接口响应字段
- 固化 Windows exe 客户端交付、Linux 服务端部署、symlink 优先且失败自动 copy 的启用口径和内置工具真实格式转换口径

### 3.2 非目标

- 不提供 Skill 发布、重新提交、版本发布表单
- 不提供审核工作台、锁单、退回、拒绝等审核 UX
- 不提供部门、用户、Skill 管理台
- 不提供 Web 管理台或独立 Web 发布入口
- 不接入企业统一身份源或外部账号系统
- 不提供 macOS、Linux 客户端或 exe 之外的安装包
- 不做完整安全扫描、依赖自动解析、依赖自动安装
- 不做风险脚本扫描
- 不做企业 IM、系统托盘通知、审计日志、数据看板
- 不做 MCP、插件、Agent/RAG 检索

## 4. 用户与使用场景

| 用户 | P1 诉求 | P1 权限口径 |
|------|---------|-------------|
| 企业普通用户 | 查找、安装、启用和更新自己可用的 Skill | 客户端只消费服务端返回的可见、可查看详情、可安装判断 |
| 管理员用户 | 作为普通使用者安装和启用 Skill | P1 不展示审核/管理入口；管理员特权进入 P2 |
| 离线用户 | 继续使用已安装 Skill，并可调整本地启用状态 | 离线期间不安装、不更新、不搜索市场 |

## 5. 范围决策

| 决策项 | P1 定稿 |
|--------|---------|
| 账号体系 | 自建账号，由管理员开通；后续考虑接入企业系统 |
| 客户端安装包 | Windows exe 安装包；暂不考虑其他客户端包 |
| 服务端部署 | Linux 服务器部署，优先简单、兼容性高的运行方案 |
| 市场数据来源 | 由服务端预置、种子数据、管理员导入或既有数据提供；Desktop 不提供发布入口 |
| 首页模块 | 展示连接状态、可更新数量、我的安装摘要、热门/推荐、最近更新、通知摘要；隐藏或不渲染 P2 发布/审核待办 |
| 导航入口 | P1 默认展示首页、市场、我的 Skill、工具、项目、通知、设置；审核、管理、发布 Skill 不进入 P1 导航 |
| 分类/标签 | P1 可展示服务端已有分类/标签；不定义分类体系，不提供治理能力 |
| 排行榜 | P1 可展示服务端直接返回的简单榜单；综合权重和最近增长榜进入 P2/P3 |
| 工具检测 | P1 使用可配置 Tool Adapter；检测方式为 Windows 注册表扫描、默认路径探测，并提供手动自定义路径兜底 |
| 内置工具 | Codex、Claude、Cursor、Windsurf、opencode、自定义目录 |
| 启用方式 | symlink 优先；失败自动降级 copy，并记录 requestedMode、resolvedMode、fallbackReason |
| 格式转换 | 对内置工具做真实格式转换；自定义目录使用通用目录复制 |
| 包大小与文件数 | 单个 Skill 包最大 5MB，包内文件最多 100 个 |
| 风险脚本扫描 | P1 不做风险脚本扫描 |
| 包校验 | 服务端必须返回包 Hash，P1 使用 SHA-256 做下载后校验；校验失败不得安装或覆盖 |
| 离线同步 | 恢复网络后仅同步本地启用/停用变更；权限、下架、版本、可安装资格以服务端为准 |

## 7. 功能需求

> 页面结构、状态、按钮、弹窗和空态错误态见 [P1 Desktop 交互规格](23_p1_interaction_spec.md)。

### 7.1 登录与连接状态

- 用户可登录 Desktop 客户端
- 顶栏展示当前用户身份、角色名称和退出入口
- 客户端展示服务端连接状态：连接正常、连接中、连接失败、离线
- 连接失败时保留本地已安装 Skill 使用能力，并提示市场搜索、安装、更新暂不可用
- 恢复连接后自动刷新市场数据、通知和可更新状态

验收标准：

- 未登录用户不能进入主应用页面
- 连接失败不影响已安装 Skill 的本地启用/停用
- 恢复连接后不自动覆盖本地 Skill，必须按更新流程处理

### 7.2 首页

P1 首页展示：

- 服务端连接状态
- 我的已安装数量
- 可更新 Skill 数量
- 最近更新 Skill
- 热门/推荐 Skill
- 通知摘要
- 快捷入口：查看我的 Skill、查看工具管理、查看项目管理、进入市场

P1 首页不展示：

- 待审核数量
- 待处理发布数量
- 被退回修改数量
- 发布 Skill
- 进入审核

### 7.3 市场

市场列表支持：

- 搜索：Skill 名称、描述、标签、作者、部门、skillID
- P1 必做筛选：部门、工具兼容性、是否已安装、是否已启用、是否仅显示我有权限查看
- P1 增强筛选：分类、风险等级、发布时间、更新时间
- 排序：综合排序、最新发布、最近更新、下载量、Star 数、搜索相关度
- 卡片展示：图标、显示名称、skillID、简短描述、作者、作者部门、当前版本、Star 数、下载量、风险等级、最后更新时间、安装/启用状态

市场详情支持：

- 完整详情：基础信息、使用信息、README/使用说明、审核与安全信息、版本信息、操作区
- 受限详情：摘要公开但无详情权限时，仅展示名称、发布人、部门、Star 数、下载量、分类/图标和无权限提示
- 操作区：安装、更新、已安装、已启用、查看已启用位置、Star/取消 Star

P1 不做：

- 申请访问 / 申请使用
- 在线发布
- 文件在线编辑
- 历史版本安装或回滚

### 7.4 我的 Skill

P1 仅交付"已安装"。

已安装列表字段：

- Skill 名称
- 当前本地版本
- 市场最新版本
- 是否有更新
- 已启用工具
- 已启用项目
- 最近启用时间
- 启用状态
- 是否受权限收缩限制

操作：

- 查看详情
- 更新
- 卸载
- 启用 / 停用
- 编辑启用范围

权限收缩规则：

- 已安装用户可继续使用当前本地版本
- 已安装但未启用用户可继续启用当前本地版本
- 权限收缩后不可更新，不可重新安装，不可新增启用范围
- 已卸载用户视为非存量用户，不能再次安装

### 7.5 安装、更新与卸载

安装流程：

1. 用户在市场点击安装
2. 客户端校验可安装资格
3. 客户端从服务端获取包地址或包标识
4. 客户端下载包到临时目录
5. 客户端校验包大小不超过 5MB、文件数不超过 100 个
6. 客户端用 SHA-256 校验包 Hash
7. 校验通过后写入 Central Store
8. 列表和详情页状态更新为已安装

更新流程：

1. 客户端发现市场最新版本高于本地版本
2. 用户点击更新
3. 若本地文件 Hash 与上次安装 Hash 不一致，提示"本次更新将直接覆盖本地内容"
4. 用户确认后下载新包
5. 校验通过后全量覆盖 Central Store
6. 已启用位置同步更新或提示用户重新启用

卸载流程：

1. 用户点击卸载
2. 客户端提示当前被哪些工具/项目引用
3. 用户确认后移除 Central Store 和所有目标目录中的链接或副本
4. 卸载后该 Skill 在本机不可继续使用

失败处理：

- 下载失败：不写入 Central Store，保留原状态
- Hash 校验失败：删除临时文件，提示重新下载
- 覆盖失败：保留原版本，提示失败路径
- 卸载部分失败：展示失败目标路径，并允许重试

### 7.6 工具与项目启用

> Tool Adapter 字段、首批内置工具和路径检测规则见 [P1 Tool Adapter 配置契约](22_p1_tool_adapter_contract.md)。

工具页要求：

- 展示工具名称、配置路径、skills 安装路径、启用状态、检测状态、已启用 Skill 数量
- 支持刷新检测工具
- 支持添加自定义工具
- 支持修改配置路径和 skills 安装路径
- 支持开启/关闭当前工具配置
- 支持对单个 Skill 启用/停用

项目页要求：

- 支持手动添加项目
- 支持设置项目路径和项目 skills 安装路径
- 支持项目启用状态管理
- 支持项目级启用/停用 Skill

路径与冲突规则：

- 同一个 Skill 每台设备本地只能有一个版本
- 工具级启用和项目级启用冲突时，项目级优先
- 工具路径与项目路径同时存在时，项目路径优先
- 启用动作若会覆盖现有文件，系统直接覆盖并提示
- 启用动作默认请求 symlink；如 symlink 失败自动降级 copy，并在本地状态和同步事件中记录降级原因

### 7.7 通知

P1 应用内通知类型：

- Skill 有新版本可更新
- Skill 权限已收缩
- 当前安装副本不可再更新
- 连接状态异常 / 恢复
- 工具或项目路径异常
- 安装、更新、卸载、启用、停用结果

通知字段：

- 通知标题
- 通知摘要
- 关联对象
- 时间
- 已读 / 未读状态

操作：

- 标记已读
- 全部已读
- 跳转关联详情或本地配置页

### 7.8 设置

P1 设置范围：

- 语言设置
- 默认语言自动识别
- Central Store 路径展示
- 基础偏好设置

P1 不提供：

- 审核偏好
- 通知渠道配置
- MCP / 插件配置

### 7.9 离线模式

离线可执行：

- 已安装 Skill 正常使用
- 切换 Skill 启用/停用
- 浏览已缓存的本地数据

离线不可执行：

- 搜索市场
- 安装/更新 Skill
- 发布 Skill
- 审核 Skill
- 修改权限范围/公开级别

恢复网络后：

- 仅同步离线期间发生的本地启用/停用变更
- 权限、下架、版本、可安装资格以服务端为准
- 若服务端有版本更新，提示用户手动更新
- 若服务端权限已收缩或 Skill 已下架，提示用户但不自动删除本地已安装副本

## 9. 验收标准

- 用户能登录并看到连接状态；断网后仍能进入本地已安装使用页面
- 市场列表能按关键字搜索，并能进入完整详情或受限详情
- 无安装权限的 Skill 不出现安装按钮或展示明确不可安装原因
- 可安装 Skill 能下载、Hash 校验、写入 Central Store，并更新本地状态
- 更新时能识别本地修改并提示覆盖
- 卸载前能列出已启用工具/项目引用，确认后移除本地副本和目标目录引用
- 工具检测失败时用户可手动添加自定义工具路径
- 项目可手动添加，并可配置项目级 skills 路径
- 启用到工具或项目时优先创建 symlink；失败时自动 copy，并能展示实际模式与降级原因
- 离线时不能搜索市场或安装/更新，但能启用/停用已安装 Skill
- 恢复网络后只同步本地启用/停用变更，不以本地状态覆盖服务端治理状态
- P1 不展示审核、管理、发布 Skill 的正式入口
