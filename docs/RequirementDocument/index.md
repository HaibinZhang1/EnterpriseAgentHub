# 企业内部 Agent Skills 管理市场（Marketplace）需求说明

## 文档信息
- 文档名称：企业内部 Agent Skills 管理市场需求说明
- 文档版本：V1.3
- 当前阶段：P1 交付边界澄清稿 / 可进入开发拆解阶段
- 适用范围：企业内网部署场景，Windows exe 桌面客户端优先，服务端部署到 Linux 服务器
- 说明：本文档聚焦前端页面、功能、交互、权限、状态、业务规则与 Skill 包规范；不展开技术栈、数据库与接口实现细节。

> 当前发布边界（2026-04-11）：Windows-first 内网生产交付以 exe 桌面客户端为唯一维护的产品/演示入口，服务端必须部署到 Linux 服务器并优先选择简单、兼容性高的方案。本需求集中的发布/审核、Web/管理台等内容代表完整产品蓝图或历史阶段上下文，本轮 Desktop 发布仅要求登录、连接状态、我的 Skill、市场/搜索/浏览、通知/状态；发布/审核 UX 延后到后续版本。

## P1 已澄清决策

- 账号：P1 使用自建账号体系，由管理员开通账号；后续再考虑接入企业系统或统一身份源。
- 客户端交付：P1 仅提供 Windows exe 安装包，暂不考虑 macOS、Linux 客户端或其他分发形式。
- 服务端部署：必须部署到 Linux 服务器，方案优先简单、稳定、兼容性高。
- 本地启用：P1 采用 symlink 优先；当权限、文件系统或安全策略导致 symlink 失败时自动降级为 copy，并记录实际模式与降级原因。
- 包限制：单个 Skill 包最大 5MB，包内文件最多 100 个；P1 不做风险脚本扫描。
- 内置目标：P1 支持 Codex、Claude、Cursor、Windsurf、opencode 和自定义目录，并对内置工具做真实格式转换。

为避免"本期"在不同历史上下文中指代不一致，本文档后续统一使用 P1 / P2 / P3 表达阶段边界。若子文档中描述了完整蓝图能力，以本阶段矩阵为准拆分交付。

| 阶段 | 目标 | 主要范围 | 明确不含 |
|------|------|----------|----------|
| P1 Desktop 使用闭环 | Windows-first 桌面端可用交付 | 登录与身份展示、连接状态、市场浏览/搜索/详情、我的已安装 Skill、安装/更新/卸载、symlink 优先且失败自动 copy 的启用/停用到工具或项目、应用内通知/状态、基础设置、离线本地使用 | 发布 Skill、审核工作台、部门/用户/Skill 管理台、Web 管理台、复杂治理报表、其他客户端安装包、风险脚本扫描 |
| P2 企业治理闭环 | 补齐企业内部发布与审批管理 | Skill 发布与版本更新、审核工作台、权限变更审核、部门/用户/Skill 管理、自建账号与角色管理、下架/上架/归档、风险标签与模板、manifest 字段枚举固化 | MCP/插件管理、RAG、智能 Agent 搜索、依赖自动安装、多维护者协作 |
| P3 生态与规模化 | 面向多工具、多团队和运营治理扩展 | MCP 管理、插件管理、Agent/RAG 检索、依赖自动解析、批量 Skill 目标管理、审计日志、数据看板、多维护者/组织级发布、企业 IM/托盘通知、组织变更迁移、多平台扩展 | 不作为 P1/P2 阻塞项 |
