# 客户端前端布局重构文档

## 1. 文档目的

本文档用于沉淀 Desktop 客户端下一阶段的前端布局重构方案，作为前端重构的直接布局依据。
本轮文档不再只定义抽象信息架构，而是明确：

- 全局布局骨架
- 页面模板归类
- 视觉基线
- 各一级入口的实际布局方案
- 弹层 / 抽屉 / 详情层的统一打开方式

本文档面向真实前端重构，默认落地入口仍为 `apps/desktop`。

本轮前端重构以以下两份内容为直接参照：

同时继续遵守以下业务与导航约束：

- 游客优先，本地能力可先使用
- 管理员能力只在在线且具备权限时显示
- 通知保留为顶栏入口，不单独展开为一级页面
- MCP、插件暂不进入正式功能范围，但允许保留预留入口

## 3. 本轮重构结论

本轮不再采用“传统后台左侧主导航 + 大长页”的组织方式，而统一收敛为：

**圆角顶栏主导航 + 单舞台工作区 + 局部侧栏切换 + 独立 Skill 详情页 + 局部悬浮窗**

统一风格方向为：

**浅色企业 AI 工作台 + 克制微动效 + 桌面级高密度操作区**

这意味着：

- 顶部是唯一一级主导航
- 页面切换以“舞台内容区切换”为主
- 社区、本地、管理允许页面内部再做局部侧栏切换
- Skill 详情使用独立页面
- 启用使用悬浮窗
- 设置、登录、诊断仍优先走弹层 / 抽屉
- 不新增底栏

## 4. 全局布局基线

## 4.1 顶栏

顶栏不是整条贴边横条，而是一个与下方工作区风格一致的圆角容器。

### 顶栏结构

```text
┌─────────────────────────────────────────────────────────────┐
│ 品牌区      社区 / 主页 / 本地 / 管理      通知 / 头像入口 │
└─────────────────────────────────────────────────────────────┘
```

### 顶栏规则

- 顶栏使用圆角容器
- 顶栏宽度与下方工作区保持同一水平约束
- 顶栏左右留白与主舞台区一致
- 顶栏不应显得比下方模块更宽
- 顶栏本身不承载复杂信息，只承载一级切换与全局入口

### 顶栏组成

#### 左侧
- Logo
- 产品名称
- 英文副标题可保留一行弱化说明

#### 中部
- 一级导航：社区、主页、本地、管理
- 管理仅管理员可见
- 一级导航采用分段切换按钮，不使用底部下划线式 tab

#### 右侧
- 通知按钮
- 头像菜单

### 顶栏交互

- 顶栏中间菜单 hover 必须有轻微变色
- 顶栏按钮 hover 不只依赖位移和阴影
- 打开 Skill 详情、设置、登录等覆盖层时，覆盖层必须在顶栏之上

---

## 4.2 舞台工作区

顶栏下方为统一舞台区，承载当前一级入口页面。

### 舞台区规则

- 主舞台区采用统一外边距
- 页面内部可以滚动，但不做传统长页式堆叠
- 每个一级页面都应先归类到一个明确模板

---

## 4.3 页面模板归类

结合 19 号视觉规范与当前原型，本轮前端重构统一采用以下模板：

### A. 概览页
适用：主页

结构：
- 左侧快捷磁贴区
- 中部 Workspace 主舞台
- 右侧推荐与热门区

### B. 市场页
适用：社区

结构：
- 左侧类型入口
- 右侧搜索 / 排序 / 标签 / Skill 卡片网格

### C. 工作台页
适用：本地

结构：
- 左侧入口切换
- 中间主列表
- 右侧当前对象详情

### D. 管理页
适用：管理

结构：
- 左侧管理入口
- 顶部统一功能栏
- 主区域列表 + 详情联动

## 5. 视觉与交互基线

本轮布局重构统一参照 `19_visual_design_system.md`，重点吸收以下规则。

## 5.1 默认主题

- 默认采用浅色主题
- 主背景为冷白 / 蓝灰浅底
- 页面允许叠加轻网格与极弱环境光
- 深色方案不作为默认交付界面

## 5.2 卡片与容器

- 顶栏、Hero、主要面板、详情面板统一使用大圆角
- 普通业务卡以白卡或半透明浅白卡为主
- 边框轻、阴影轻，不使用厚重拟物阴影

## 5.3 Hover 规则

- hover 不只做轻微位移
- 必须补充轻微变色
- 浅色主题默认使用主色 `3%` 到 `8%` 的浅染色背景

适用对象：
- 顶栏导航按钮
- 卡片
- 列表行
- Ghost / Secondary 按钮
- 左侧切换按钮

## 5.4 图标与图片

本轮原型已经收敛为：

- Skill 列表不展示封面图片
- 社区 Skill 卡不展示大图封面
- 本地 Skill 列表不展示图片
- Skill 详情页不展示大图
- 统一改为 Skill 名称首字母或首字母组合图标

因此前端重构时，Skill UI 默认应采用：

- 小尺寸字母图标：用于社区卡片、行列表
- 中尺寸字母图标：用于本地列表、信息行
- 大尺寸字母图标：用于详情抽屉、Inspector 主标识

## 5.5 顶部统一功能栏

社区、本地、管理三类页面统一采用：

**左侧导航栏 + 顶部功能栏 + 下方内容区**

其中顶部功能栏规则为：

- 左侧使用与左侧导航栏一致的“图标 + 名称”组件
- 左侧显示当前被选中的功能说明，例如：`Skills`、`工具`、`项目`、`审核`
- 右侧可为空；仅在当前页面确实存在主操作时再展示按钮，例如：`扫描`
- 不再重复使用“Local Workspace / 本地”这类与一级导航含义重复的标题
- 社区的搜索和标签不放进这个功能栏，而是放到下方独立区域

## 6. 一级入口布局定义

## 6.1 主页

### 页面定位

主页是桌面客户端的中心探索入口，不承载继续处理、热榜或本地入口等辅助区块，只负责把用户直接带入提问与探索。

### 页面骨架

```text
┌──────────────────────────────────────────────────────────────┐
│                     Agent 探索主舞台                         │
│                     标题 + 提问编辑框                         │
└──────────────────────────────────────────────────────────────┘
```

### Agent 探索主舞台

必须包含：

- 标题：`Agent 探索`
- 上移后的提问编辑框

主页主舞台居中，但内容本身应更靠上，不做垂直完全居中。
主页不再承载左右侧工作入口、继续处理或热榜区块。

### 提问编辑框

主页输入区不再使用单行搜索框，而改为更像桌面 Agent 的文本编辑框。

结构要求：

- 上方为多行输入区域
- placeholder 提示用户可提问、添加文件、输入命令、使用技能
- 下方为工具条，可承载权限、模型、模式、语音、发送等按钮
- 底部可保留上下文 pill，例如当前工作区、分支、环境
- 整体外观接近编辑器 / 对话输入面板，而不是搜索栏

### 主页设计原则

- 主页只回答“从哪里开始探索”
- 主舞台必须尽量纯净
- 不保留工作入口区
- 不保留继续处理区
- 不保留热榜区
- 不把主页做成营销页

---

## 6.2 社区

### 页面定位

社区承担 Skill 发现、搜索、筛选、排序、查看详情、安装与发布入口。

### 页面结构

当前重构口径不再采用“单列大摘要列表”，而采用：

**左侧社区入口 + 顶部统一功能栏 + 对应内容区**

```text
┌──────────────┬──────────────────────────────┬──────────────┐
│ 社区入口      │ 顶部统一功能栏                │ 顶部留白      │
│ Skill        ├──────────────────────────────┼──────────────┤
│ MCP 预留      │ Skills: 搜索 / 标签 / 卡片网格 │ 热榜侧栏      │
│ 插件 预留     │ 发布: 单列发布表单             │              │
│ ──────────   │ 我的: 我发布的 Skill 与详情      │              │
│ 发布          │                              │              │
│ 我的          │                              │              │
└──────────────┴──────────────────────────────┴──────────────┘
```

### 左侧入口区

保留五个入口：

- Skill
- MCP（预留）
- 插件（预留）
- 发布
- 我的

规则如下：

- 当前版本仅开发 `Skill`
- `MCP` 与 `插件` 入口允许展示，但仅显示占位态
- 占位态必须明确标注“预留 / 后续开放”
- `发布` 与 `我的` 作为作者入口，需与上方发现入口使用横线分隔
- `发布` 进入发布表单
- `我的` 展示我发布的 Skill、状态、公开级别、授权范围、文件预览与上下架 / 归档动作

### 顶部统一功能栏

必须包含：

- 左侧：图标 + 当前子入口名称
- 右侧可为空；社区中的“发布”改由左侧入口承载

### 搜索与筛选区

搜索与标签独立成一个单独区域，位于顶部功能栏下方。

- 第一行：搜索框 + 排序
- 第二行：标签按钮行

### Skill 卡片网格 + 热榜侧栏

社区页中间主区继续使用 Skill 卡片网格，右侧新增独立热榜侧栏。

每个 Skill 卡片展示：

- 小型名称首字母图标
- Skill 名称
- 描述
- 分类标签
- 状态标签
- 作者 / 部门 / 版本
- 下载量 / Star

### 社区设计原则

- 社区是“浏览与发现”工作台，不是管理页
- 发现入口与作者入口都留在社区内部，不再拆成右上角发布按钮
- 卡片网格应服务于比较与扫描
- 卡片不展示大图封面
- 社区卡片不在卡片底部堆叠“启用 / 查看完整详情”按钮
- 用户点击整张卡片后进入独立 Skill 详情页
- 热榜放在社区右侧，不再放在主页
- 热榜采用更简洁的榜单行格式，每行只展示字母图标、名称、单个分类标签、Star、下载量

## 6.3 本地

### 页面定位

本地页以 Skill 管理为核心，聚焦：

- 已安装 Skill
- 本机启用状态
- 更新
- 异常
- 卸载

### 页面结构

本地页采用标准工作台结构：

```text
┌──────────────┬──────────────────────────────┬──────────────┐
│ 本地入口      │ 顶部统一功能栏                │ 顶部留白      │
│ Skills       ├──────────────────────────────┼──────────────┤
│ 工具         │ 搜索 / 筛选 / 本地主列表       │ 当前详情      │
│ 项目         │                              │              │
└──────────────┴──────────────────────────────┴──────────────┘
```

### 左侧入口区

保留三个入口：

- Skills
- 工具
- 项目

这三者不是头部轻量按钮，而是本地页内部的一级切换入口。

### 顶部统一功能栏

- 左侧显示当前选中的功能，例如 `Skills`、`工具`、`项目`
- 右侧显示主功能按钮，例如 `扫描`
- 不再重复展示“本地 / Local Workspace”这一类重复标题

### 中间主列表区

当选择 `Skills` 时，显示：

- 搜索框
- 状态筛选
- 本地 Skill 列表

每个本地 Skill 行展示：

- 名称首字母图标
- 名称
- 描述
- 作者 / 部门 / 本地版本
- 当前状态
- 更新 / 启用 / 卸载等操作
- 目标数量
- 更新时间
- 异常摘要（如有）

当选择 `工具` 或 `项目` 时，中间区域切换为对应对象列表。

### 右侧详情区

用于展示当前选中对象：

- 当选中 Skill：展示大号字母标识、说明、启用位置、当前提示、主操作
- 当选中工具：展示路径、状态、重新扫描等动作
- 当选中项目：展示项目路径、启用管理入口等信息

本地页保持“Skill 列表 + 右侧简介”这种主从结构不变。
当用户从右侧简介点击“查看完整详情”时，再跳转到独立的 Skill 详情页。

### 本地设计原则

- 主焦点始终是“当前本地生效情况”
- 工具和项目可达，但不应压过 Skill 视角
- 本地页默认保持“列表 + 当前详情”联动

## 6.4 管理

### 页面定位

管理页是管理员专属工作区，用于整合原有管理员分散入口。

### 可见性规则

- 仅管理员显示
- 仅在在线且具备权限时显示
- 普通用户完全不显示该入口

### 页面结构

管理页采用：

**左侧管理入口 + 顶部统一功能栏 + 主区域列表与详情联动**

```text
┌──────────────┬──────────────────────────────────────────────┐
│ 管理入口      │ 顶部统一功能栏                                 │
│ 审核         ├──────────────────────────────────────────────┤
│ Skills       │ 当前模块的列表与详情联动                         │
│ 部门         │                                                │
│ 用户         │                                                │
└──────────────┴──────────────────────────────────────────────┘
```

### 管理入口

固定四类：

- 审核
- Skills
- 部门
- 用户

### 顶部统一功能栏

- 左侧显示当前被选中的功能，例如：`审核`、`Skills`、`部门`、`用户`
- 右侧显示当前模块的主功能按钮，例如：`新增部门`

### 各模块说明

#### 部门
- 部门目录
- 当前部门详情
- 成员数量 / Skill 数量 / 负责人

#### 用户
- 用户列表
- 用户详情
- 角色调整 / 冻结等动作

#### Skills
- Skill 管理列表
- 上下架、归档、查看详情

#### 审核
- 审核队列
- 审核详情
- 文件预览
- 审核动作

### 管理设计原则

- 管理相关能力全部收口到一个一级入口中
- 页面内部再通过左侧入口切分任务
- 管理页允许更高密度，但结构必须稳定

## 7. 全局弹层与覆盖层规则

## 7.1 Skill 详情

- Skill 详情不再通过右侧滑出层打开
- Skill 详情改为独立的舞台页面
- 从社区卡片或本地右侧简介点击“查看完整详情”时，进入独立详情页
- 顶栏仍保留，但详情内容不再与列表同屏挤压
- 详情页内部不展示 Skill 大图，改为大号字母标识

详情页内容至少包含：

- Skill 名称
- 描述
- 作者 / 部门 / 版本 / 状态
- Star / 下载量 / 兼容工具 / 最近变化
- 已启用位置
- 主操作区

## 7.2 启用

- Skill 的“启用 / 调整启用”不再跳详情层
- 改为悬浮窗形式打开
- 悬浮窗在当前列表上下文之上显示，不打断当前页面结构
- 悬浮窗内容参考“目标选择”模式，以目标卡片网格为主，而不是简单 tag 列表
- 悬浮窗内至少包含：
  - 当前 Skill 名称
  - 当前版本号
  - 当前已启用数量
  - 可启用目标卡片列表
  - 每个目标的名称、路径、状态说明
  - 应用目标 / 添加自定义工具 / 添加项目 等动作

## 7.3 设置入口

- 设置从头像菜单进入
- 设置使用模态窗
- 不再占据一级导航位置

## 7.4 登录入口

- 登录使用模态窗
- 用户触发需要远端权限的动作时再弹出
- 登录成功后回到原上下文继续操作

## 7.5 诊断 / 工具 / 项目扩展能力

- 诊断、工具、项目允许用抽屉承载补充信息
- 不为了这些能力新开一级页面

## 8. 与现有能力的映射关系

| 重构后入口 | 当前能力来源 |
| --- | --- |
| 主页 | `home` |
| 社区 | `market` |
| 本地 | `my_installed` + 工具 / 项目 / 诊断相关能力 |
| 管理 | `review` + `admin_departments` + `admin_users` + `admin_skills` |

补充说明：

- `发布中心` 不再作为一级主导航单独暴露，吸收到社区左侧入口中的 `发布 / 我的`
- `发布 Skill` 页面采用单列长表单，按顺序展示：上传区、Slug、显示名称、描述、版本号、公开级别、变更说明
- `目标管理` 不再作为一级主导航单独暴露，吸收到主页快捷卡片和本地页结构中
- 通知继续由顶栏按钮承载

## 9. 前端实现约束

前端重构时必须遵守以下约束：

1. 所有页面统一遵守 `19_visual_design_system.md`
2. 当前可直接参照 `layout-prototype copy` 作为布局与交互原型
3. 顶栏必须是圆角容器，并与主工作区宽度对齐
4. hover 必须是“轻微位移 / 提亮 + 轻微变色”的组合
5. Skill 相关列表与详情默认使用首字母图标，不再依赖封面图片
6. 详情抽屉、登录、设置、诊断等覆盖层必须在顶栏之上
7. 不再回退到“左侧一级导航 + 长页面堆叠”的老布局

## 10. 本轮文档结论

本次前端布局重构收敛为以下稳定方案：

- 一级业务入口固定为：主页、社区、本地、管理
- 顶栏采用与工作区一致的圆角容器，不再铺满全宽
- 主页采用“左侧快捷磁贴 + 中部 Workspace + 右侧继续处理”
- 社区采用“左侧类型入口 + 搜索排序 + 标签行 + Skill 卡片网格 + 右侧热榜”
- 本地采用“左侧入口切换 + 主列表 + 右侧当前详情”
- 管理整合管理员能力，通过左侧入口切换部门、用户、Skills、审核
- Skill 详情改为独立页面
- Skill 启用改为悬浮窗
- 设置、登录、诊断继续走覆盖层 / 抽屉 / 模态
- Skill 默认采用名称首字母图标，不再依赖图片
- 顶部右侧保留通知与头像菜单
- 不展示底栏

该文档即为下一阶段前端重构的布局依据，后续低保真、视觉稿与实际代码实现均以本文档 + `19_visual_design_system.md` + `layout-prototype copy` 三者共同收敛。
