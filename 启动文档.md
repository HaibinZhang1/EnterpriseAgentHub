# EnterpriseAgentHub 启动与联调手册

本文档整理了当前仓库在 Windows + WSL2 环境下的实际启动方式、重启步骤、联调检查命令，以及桌面端安装包构建与验证方式。

当前已验证通过的组合如下：

- Windows 原生环境运行桌面端和安装包构建
- WSL2 Linux 环境运行 Docker 化服务端
- Docker Desktop 提供 Windows 与 WSL2 的 Docker 引擎
- 当前联调示例服务地址使用 `http://127.0.0.1:3000`

## 目录结构

- 后端 API：`apps/api`
- 桌面端：`apps/desktop`
- 服务端 Compose：`infra/docker-compose.prod.yml`
- 服务端环境模板：`infra/env/server.env.example`
- 部署检查脚本：`deploy/server-check.sh`
- 部署启动脚本：`deploy/server-up.sh`

## 前置条件

Windows 侧建议先确认：

```powershell
node -v
npm -v
rustc -V
cargo -V
docker version
docker compose version
wsl.exe -l -v
```

如果 `docker version` 报错找不到 `dockerDesktopLinuxEngine`，先启动 Docker Desktop：

```powershell
Start-Process 'C:\Program Files\Docker\Docker\Docker Desktop.exe'
```

WSL2 侧建议确认：

```powershell
wsl.exe -d Ubuntu --cd /mnt/g/train/EnterpriseAgentHub bash -lc 'docker version && docker compose version'
```

## 首次安装依赖

在仓库根目录执行：

```powershell
npm install
```

如果只需要补桌面端依赖，也可以执行：

```powershell
npm install --workspace @enterprise-agent-hub/desktop
```

## 服务端启动

### 1. 准备环境文件

首次启动前复制环境模板：

```powershell
Copy-Item infra\env\server.env.example infra\env\server.env
```

当前已验证可用的关键配置如下：

```env
API_PORT=3000
API_IMAGE=enterprise-agent-hub-api:0.1.0-p1
COMPOSE_PROJECT_NAME=enterprise-agent-hub
DATABASE_URL=postgresql://eah:change-me@postgres:5432/enterprise_agent_hub
REDIS_URL=redis://redis:6379/0
MINIO_ENDPOINT=minio
MINIO_PORT=9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=change-me-minio-secret
JWT_SECRET=change-me-before-deploy
```

### 2. 预检查

按仓库约定，优先通过 `bash` 调用脚本，不假设执行权限：

```powershell
wsl.exe -d Ubuntu --cd /mnt/g/train/EnterpriseAgentHub bash -lc 'bash deploy/server-check.sh'
```

### 3. 启动服务端

推荐命令：

```powershell
wsl.exe -d Ubuntu --cd /mnt/g/train/EnterpriseAgentHub bash -lc 'bash deploy/server-up.sh'
```

如果需要先在本地重建 API 镜像，再启动 Compose：

```powershell
wsl.exe -d Ubuntu --cd /mnt/g/train/EnterpriseAgentHub bash -lc 'docker build -t enterprise-agent-hub-api:0.1.0-p1 -f apps/api/Dockerfile .'
wsl.exe -d Ubuntu --cd /mnt/g/train/EnterpriseAgentHub bash -lc 'bash deploy/server-up.sh'
```

如果脚本链路异常，也可以直接在 Windows 侧使用 Compose：

```powershell
docker compose --env-file infra/env/server.env -f infra/docker-compose.prod.yml -p enterprise-agent-hub up -d postgres redis minio minio-init api-migrate api-seed api
```

### 4. 查看运行状态

```powershell
docker compose --env-file infra/env/server.env -f infra/docker-compose.prod.yml -p enterprise-agent-hub ps
```

期望状态：

- `postgres` healthy
- `redis` healthy
- `minio` healthy
- `api` healthy

### 5. 服务端接口验证

健康检查：

```powershell
Invoke-RestMethod http://127.0.0.1:3000/health
```

已验证返回类似：

```json
{
  "status": "ok",
  "api": "ok",
  "postgres": "ok",
  "redis": "ok",
  "minio": "ok"
}
```

登录验证：

```powershell
$body = '{"phoneNumber":"13800000001","password":"demo123"}'
Invoke-RestMethod http://127.0.0.1:3000/auth/login -Method Post -ContentType 'application/json' -Body $body
```

桌面启动上下文验证：

```powershell
Invoke-RestMethod http://127.0.0.1:3000/desktop/bootstrap
```

当前已验证通过的最终服务端地址：

- `http://127.0.0.1:3000`

如果 `127.0.0.1:3000` 不通，再改用 WSL2 实际 IP：

```powershell
wsl.exe -d Ubuntu bash -lc "hostname -I | awk '{print \$1}'"
```

## 桌面端开发联调

### 1. Web 开发模式

```powershell
npm run dev --workspace @enterprise-agent-hub/desktop
```

### 2. Tauri 开发模式

优先使用这一条做真实桌面联调：

```powershell
npm run tauri:dev --workspace @enterprise-agent-hub/desktop
```

说明：

- 桌面端会优先读取 `VITE_DESKTOP_API_BASE_URL`
- 如果未设置，就按登录弹窗里输入的服务地址连接，不再内置某个固定端口
- 如果需要预填地址，请显式设置 `VITE_DESKTOP_API_BASE_URL=http://127.0.0.1:3000`

## 常用重启命令

### 仅重启服务端

```powershell
docker compose --env-file infra/env/server.env -f infra/docker-compose.prod.yml -p enterprise-agent-hub down
docker compose --env-file infra/env/server.env -f infra/docker-compose.prod.yml -p enterprise-agent-hub up -d postgres redis minio minio-init api-migrate api-seed api
```

```powershell
Get-Process enterprise-agent-hub-desktop -ErrorAction SilentlyContinue | Stop-Process -Force
Start-Process 'C:\Users\ZhangHB\AppData\Local\Enterprise Agent Hub\enterprise-agent-hub-desktop.exe'
```

### 一次性重启整套联调环境

```powershell

```

## 常见问题

### Docker Desktop 管道不存在

报错示例：

```text
failed to connect to the docker API at npipe:////./pipe/dockerDesktopLinuxEngine
```

处理方式：

- 先启动 Docker Desktop
- 再重新执行 `docker version`
- 确认 Server 段已正常显示

### API 因 Redis 未就绪退出

如果日志出现 `ioredis ... ETIMEDOUT`：

- 先确认 `postgres`、`redis`、`minio` 都已正常启动
- 再重新执行 `docker compose ... up -d ...`

### 127.0.0.1 无法访问

优先排查：

- Docker Desktop 是否启动
- Compose 端口映射是否存在
- `api` 容器是否 healthy
- Windows 防火墙或端口占用问题

必要时退回 WSL2 IP 访问。
